About ROI Module Administration

The Administration function of the Release of Information (ROI) Module provides the ability to configure and maintain options related to the processing and tracking of ROI requests.  Users with Administration rights can:

  • Maintain reasons for requests, statuses, and invoice adjustments.

  • Define requestor types.

  • Maintain letter templates and standard notes that users can add to letters and invoices.

  • Define billing and shipping options related to ROI requests, including options for items such as billing tiers, fee types, delivery methods, and payment methods.

  • Configure whether to mask or display SSNs and MRNs (medical record numbers).

  • Flag specific document types as disclosure documents and Meaningful Use documents.

  • Manage output jobs and queues.

  • Maintain settings for external clinical systems that are integrated with the ROI Module.

 

The following items apply globally for ROI administrators:

  • Any changes you make apply going forward; they are not retroactive.

  • Any options you change apply across the enterprise; they are not facility specific.

  • Data you enter (such as names and descriptions for items) is not case sensitive.

  • Users who are logged in when you make changes must log out and then log back in for the changes to take effect.

  • The data on many ROI forms is in grid format. You can sort the data in alphabetical order or reverse-alphabetical order by clicking the column header.

Note:  With the exception of configuring output settings, all administration features can be accessed from the left pane of the Administration tab.  All features, including configuring output settings, can be accessed from the Administration menu bar

 

ROI security rights

ROI security rights (permissions) are granted in the MPF Workstation user profile and apply across the enterprise. The ROI Module and the MPF Workstation application follow the same rules for document searches, document access, and document display based on your MPF Workstation permissions. This means the document tree behaves identically in both applications.

When you grant or revoke security rights for users who are logged in, the old settings apply until they log out and log back in.

For additional information about granting security rights, see the MPF Workstation System Administrator's Guide.

Administrator rights

The following security rights are required to be an ROI administrator:

  • ROI Access Application

  • ROI Administration

End-user rights

Administrators can grant rights to other users so they can perform the functions of their role.

User rights include:

  • ROI Access Application -- This basic right is required to access the ROI Module.

  • ROI Create Request

  • ROI Modify Request

  • ROI Delete Request

  • ROI VIP Status -- This permission provides visibility to the medical records of patients with VIP status.

  • ROI Cancel Request

  • ROI Pend Request

  • ROI Deny Request

  • ROI Adjust Charges

  • ROI Post Payments

  • ROI Reports -- This permission is required to generate ROI reports.

  • ROI Print/Fax/Email -- This permission is required to print, fax or email ROI documents and letters.

  • ROI Export to PDF -- This permission is required to save ROI documents and letters to a file. This option is also called Save As File.

 

Auditing

Auditing options are configured in McKesson Patient Folder Workstation. For information, see the MPF Workstation System Administrator's Guide.
