Change request status

This function enables an authorized user to change the request status for a request and/or status reason(s). Some status options are restricted and cannot be manually assigned, but can be changed.

You cannot manually assign the following statuses:

  • Auth-Received. You cannot manually assign this status. The system assigns this status when you create a skeleton request by scanning, indexing, and releasing an authorization request form. A workflow thread automatically assigns the status.

If you have permissions, you can change this status to Logged, Pended, Denied, or Canceled, or to Completed (if at least some documents have been released).

  • Auth-Received Denied. You cannot manually assign this status. The system assigns this status when you create a skeleton request by scanning, indexing, and releasing an invalid Auth-Received request form, which has a document type that is different from valid authorizations. A workflow thread automatically assigns the status.

If you have permissions, you can change this status to Logged, Pended, or Canceled, or to Completed (if at least some documents have been released).

  • Pre-Billed. You cannot manually set this status. The system assigns this status when you click the Pre-Bill button on the Billing & Payment form.

If you have permissions, you can change this status to Logged, Pended, Denied, or Canceled (if you have permissions), or to Completed (if at least some documents have been released).

A special situation exists when a request displays a status of "Unknown." In those cases, the system cannot automatically determine the status of the request. You must ascertain the status and manually assign it to the request.

Note: You cannot set the status to Denied or Canceled if even one document has been released for the request. If you set the status to Canceled or Denied, no further changes (such as adding or removing documents) can be made to the request. However, users will be able to add comments, generate letters, or change the status again. If you cancel or pend the request, the system automatically writes off any balance due.

  1. Find a request.

  2. On the Request Information form, click Update Request Status.    

  1. Complete the Update Request Status dialog box.

  1. On the Request Information form, click Save to complete the event.
