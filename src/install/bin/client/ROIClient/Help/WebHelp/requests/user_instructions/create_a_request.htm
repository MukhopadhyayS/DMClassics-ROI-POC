Create a request

You can create a request by either scanning a paper document (recommended) or by using the Create Request function.

To create a request by scanning a letter or a form

This is the recommended method.

Prerequisites

This method assumes that:

  • You are an experienced McKesson Patient Folder user who knows how to use the DCS/QCI Workstation module.

  • You have received or generated a paper document (such as a letter or a form) that defines the request.

  • Your site has configured one MPF document type per facility code set to use with valid authorization requests and one for invalid authorization requests.

  • Your site has configured a day folder to associate with authorization requests for non-MPF patients.

This method associates the authorization with at least one patient encounter if the patient is found in the MPF database. If the patient is not found, the authorization is indexed to a day folder. Whether the authorization is for an MPF patient or a non-MPF patient, the result is the same. The system automatically creates a skeleton ROI request.

  1. Using the McKesson Patient Folder DCS/QCI Workstation module, scan the paper document.

  2. Associate the scanned document with the document type used by your site to support scanned authorizations (either valid or invalid).

  3. Index the document to either a patient encounter (for an MPF patient) or a day folder (for a non-MPF patient).

    Day folder identifiers are specified for your site when the ROI Module is installed. They usually use a dummy facility code and use an MRN formatted as yyyymm and Encounter formatted as yyyymmdd.

  1. Release the document to the McKesson Patient Folder system.

    • If no indexing errors occur and the document is successfully released, the system creates an ROI request, assigns a unique request ID, and sets the request status to "Auth-Received" or "Auth-Received Denied." You can then access the request from the ROI Module for further processing. (See Find a request and select the "Auth-Received" or "Auth-Received Denied" request status as the search criteria.)

    • If indexes fail system validation, the system does not create an ROI request, and you must use the MPF Index Correction Utility to make index corrections.

    • If indexes pass system validation but are wrong, you must use the ROI Module to delete the request, and then use the DCS/QCI Workstation module to re-scan the authorization document and associate it with the correct patient encounter or day folder.

To create a request using the Create Request function

This function allows you to manually create a request within the ROI module. This method may be appropriate when, for example, records are requested by an established, frequent requestor for whom an authorization document is waived. You can create a request from several points in the module, including the Patients tab, the Requestors tab, and the Requests tab.
