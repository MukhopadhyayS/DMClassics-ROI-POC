Local File option

The Create New Attachment dialog box displays the fields listed below only when you select Local File at the Attachment Location field.

The dialog box allows you to select a qualified file that currently resides in a location on your local hard drive, a network drive, or on transportable media like a thumb drive and attach it to the patient's record. When you attach the file, the system copies it to a secure location in the McKesson Patient Folder system and lists it in the patient's document tree.

Item descriptions

If Local File is selected, the Create New Attachment dialog box contains the following items.

Document and patient identifiers

For existing attachments, all fields display current information and, except for Facility, can be edited until the file is released. For new attachments, all fields except Facility are blank and allow you to enter data. This group contains the following items:

  • Title
    This is the title you want to give to the attachment and is displayed in the document tree. Free-form.

  • Date of Service
    This is the date that patient care occurred.

  • Document Source
    This is the name of the assigning authority or the source clinical system (for example, McKesson Paragon).

  • Facility
    The facility where patient care occurred (cannot be edited).

  • Enc/Acct
    The encounter number or account number for this patient.

Raw (XML) group

For existing attachments, these fields cannot be edited. For new attachments, fields are blank. This group contains the following items:

  • Raw(XML) File
    The name of the encrypted file you want to associate with the currently selected patient. Currently, only the XML raw file type is supported. Either enter the file path and name, or click the Select button to browse for a valid file.

  • Page Count
    Currently, XML files are defined as always having 1 page.

Formatted (PDF, DOC, etc.) group

For existing attachments, these fields cannot be edited. For new attachments, fields are blank. This group contains the following items:

  • Formatted File
    The name of the human-readable file you want to associate with the currently selected patient.  Either enter the file path and name, or click the Select button to browse for a valid file.

  • Page Count
    The number of pages in the document. Required. Populated automatically when the module generates a PDF file from the raw file. Otherwise, must be manually entered.

Comment

Free-form comment. For existing attachments, can be edited even after the file is released.

Save

Associates the file(s) with the currently selected patient and saves them to the location configured for your facility.

Cancel

Discards unsaved entries and returns you to the parent form.
