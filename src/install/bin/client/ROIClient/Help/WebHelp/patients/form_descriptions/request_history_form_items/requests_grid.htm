Requests grid

Displays requests for the patient selected on the Find Patient form and allows you to a select request.

Can be sorted on all columns.  The default sort order is ascending Receipt Date and then ascending Status.

Note: If a request contains records from facilities that you are not allowed to access, the detail pane lists the request in red text and masks data in the following columns: Requestor Name and Requestor Type. If you select the request, the system disables the View/Edit Request button.

The grid contains the following information:

  • Last Updated
    Used as the default sort in most recent to oldest order; format is mm/dd/yyyy.
    The date when the request was last updated.

  • Facility
    The facility where patient care occurred.
    Used as the default secondary sort in ascending alphabetical order.

  • Updated By
    The user who last updated the request. Initial sort order is alphanumeric ascending.
    Format is: LastName, FirstName MiddleInitial

  • Status
    The status of the request: Auth-Received, Canceled, Completed, Denied, Logged, Pended, Pre-Billed.

  • Requestor Name
    Displays the requestor name for the request. Initial sort is alphanumeric ascending.

  • Requestor Type
    Displays the requestor's type. Requestor Types are maintained in the ROI application and identify the category of requester, such as attorneys, employers, or insurance. Initial sort is alphanumeric ascending.

  • Request Id
    Displays the ID of the request. Initial sort order is alphanumeric ascending.

  • Receipt Date
    Displays the date the request was received; format is mm/dd/yyyy.
    Initial sort order is oldest to most recent date.

  • Balance
    Displays the total charges (in dollars) for the request. Initial sort order is numeric ascending.
