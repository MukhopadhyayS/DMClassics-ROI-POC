About ROI Module Administration

The Administration function of the Release of Information (ROI) Module provides the ability to configure and maintain options related to the processing and tracking of ROI requests. Users with the ROI Administration right can:

  • Maintain standard request reasons, request status reasons, and reasons for adjusting the invoice amount for requests.

  • Define requestor types.

  • Maintain letter templates and standard notes that users can add to letters and invoices.

  • Define billing and shipping options related to ROI requests, including options for items such as billing tiers, fee types, delivery methods, and payment methods.

  • Configure whether to mask or display SSNs and MRNs (medical record numbers).

  • Define document types as disclosure documents.

  • Manage output jobs and queues.

With the exception of configuring output settings, all administration features can be accessed from the left pane of the Administration tab. All features, including configuring output settings, can be accessed from the Administration item on the menu bar.

ROI security rights

ROI security rights (permissions) are granted in the MPF Workstation user profile and apply across the enterprise.

When you grant or revoke security rights for users that are logged in, the old settings apply until they log out and log back in.

For additional information about granting security rights, see the MPF Workstation System Administrator's Guide.

Note:  The ROI security rights listed in the MPF Workstation Administrator's Guide do not apply to Release 13.5. They apply to prior releases. The rights listed below apply to the current release.

Administrator rights

The following security rights are required to be an ROI administrator:

  • ROI Access Application

  • ROI Administration

End-user rights

Administrators can grant the following rights to other users so they can perform the functions of their role:

  • ROI Access Application -- The basic right required to access the ROI Module.

  • ROI Create Request

  • ROI Modify Request

  • ROI Delete Request

  • ROI VIP Status -- Provides visibility to the medical records of patients with VIP status.

  • ROI Cancel Request

  • ROI Pend Request

  • ROI Deny Request

  • ROI Adjust Charges

  • ROI Post Payments

  • ROI Reports -- Required to generate ROI reports.

  • ROI Print/Fax -- Required to print or fax ROI documents and letters.

  • ROI Export to PDF -- Required to save ROI documents and letters to a file. This option is also called Save As File.

Facts for administrators

The following items apply globally for ROI administrators:

  • Any changes you make apply going forward; they are not retroactive.

  • Any options you change apply across the enterprise; they are not facility specific.

  • The data you enter (such as the names and descriptions for items) are not case sensitive.

  • Users that are logged in when you make changes must log out and then log back in for the changes to take effect.

  • The data on many ROI forms is in grid format. You can sort the data in alphabetical order or reverse-alphabetical order by clicking the column header.

Auditing

Auditing options are configured in McKesson Patient Folder Workstation. For information, see the MPF Workstation System Administrator's Guide.
