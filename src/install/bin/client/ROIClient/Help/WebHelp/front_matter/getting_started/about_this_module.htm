About this module

The McKesson Patient Folder Release of Information (ROI) Module provides a tool for managing all requests for copies of medical records, including a billing component.  The ROI Module enables you to select and release patient information to entities who request patient records, and to easily generate letters, billing notices, and invoices for that service. You can manage and account for the release of both MPF and non-MPF records.

Release of Information features enable users to:  

  • Create and maintain standard ROI requestor types (an administrative function)  

  • Define default profiles of requestors with frequent requests (an administrative function)  

  • Create and maintain multiple billing profiles and charge types (an administrative function)  

  • Define packets of documents for release  

  • Pend, deny, or cancel a request  

  • Print or fax release documents, or save documents to a file  

  • Generate ROI reports based on a variety of selection and sort criteria; productivity and billing reports

  • Generate cover letters, pre-bills, and invoices, as well as other letters, such as denial letters or pend letters  

  • Print an ROI Disclosure Report for a patient  

  • Automatically create work assignments from requests

The module includes advanced features that enable users to more effectively handle ROI requests.

Advanced features allow you to:

  • Add requests for patients who do not exist in the MPF database, to search for the request using a reference number, and to view the request history

  • Reject requests for a patient who is not in the registration system

  • Charge for electronic documents and hardcopy documents in the same tiered fee structure

  • Post multiple adjustments to the request balance

  • Restrict writing off balances to only a supervisory level

  • Indicate the type of payment that was received (for example, check, money order, cash, and so on)

  • Add pages generated from another system and/or from hardcopy, and record each type of document for Accounting of Disclosures

  • Generate an Accounting of Disclosures Report as required by HIPAA legislation that includes all data released, including those documents and/or encounters that were added as free-text in the request. Users can decide whether to include only TPO (Treatment/Payment/Healthcare Operations) requests, only non-TPO requests, or all requests in the report

  • Select a shipping method

  • Search for requestors and associated requests

  • Customize letters and invoices

  • Select the desired letter without requiring the letter be directly tied to a status

  • Generate denial and rejection letters, including reasons (standard or free-form)

Auditing

The ROI Module can maintain an audit trail history of releases and other ROI activity. Audit trail events are configured in the MPF Workstation application for each facility.

Record views in ROI

The Patient Records form and the Patient Information form allow you to select record views for subsets of documents associated with the patient’s encounters. Record views identify standardized packets of documents for specialized functions such as coding, business office, or clinical. Record views are maintained by the McKesson Patient Folder system administrator. Only one record view can be associated with a request as a default record view. All other record views available to the user are available for selection.

ROI requests on file

The Request History form contains a list of all existing requests for a selected patient. You can select a request to view its details, and can add, change, or delete a request.
