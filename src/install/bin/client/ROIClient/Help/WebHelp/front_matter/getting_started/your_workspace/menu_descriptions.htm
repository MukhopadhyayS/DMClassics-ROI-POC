Menu descriptions

The ROI Module's menu bar contains cascading menus that reflect functions available on the window tabs. Your user permissions dictate which menus and menu items are enabled. Most menus and menu items are also available through a corresponding tab and options in the left pane.

The menu bar contains these menus:

  • Security

    The Security menu has no corresponding tab on the ROI window. The menu contains these options:

    • Change Password provides access to the Change Password dialog box.

    • Monitor Jobs allows you to view output jobs currently in the output queues. Authorized users can hold and release queue activity, and can resubmit, pause, resume, and cancel specific output jobs.

    • Logoff allows you to log off without closing the module window.

    • Exit allows you to log off and close the module window.

     

  • Patients

    The Patients menu contains these options:

    • Find Patient allows you to search for a specific patient using various search criteria like name, date of birth, facility, and MRN or account numbers.

    This option is always enabled. The remaining menu options become enabled only after you select a patient and view information.

    • Patient Information displays demographic information for the current patient. You can only view information for an MPF patient. You can either view or edit information for a non-MPF patient.

    • Patient Records provides access to record views and lists records associated with each view for the current patient. It also allows you to display document in a document viewer window.

    • Request History displays a list of previous requests associated with the current patient.

    • Create Request with Patient allows you to create an ROI request for the current patient.

    • Create Request with Patient Requesting allows you to create an ROI request for the current patient and specify that it is the patient who is requesting record.

     

  • Requestors

    The Requestors menu contains these options.

    • Find Requestor allows you to search for a specific requestor using various search criteria like name, type, status, and patient date of birth or SSN.

    This option is always enabled. The remaining menu options become enabled only after you select a requestor and view information.

    • Requestor Information displays demographic information for the current requestor and allows you to either view or edit information.

    • Request History displays a list of previous requests associated with the current requestor and allows you to select a specific request to view details.

    • Create Request with Requestor allows you to create an ROI request for the current patient and specify that it is a non-patient requestor who is requesting records.

     

  • Requests

    The Requests menu contains these options:

    • Find / Create Request allows you to search for a specific request using various search criteria like patient name, date of birth, facility, MRN or account numbers, and request status, name, type, receipt date, invoice number, or ID.

    This option is always enabled. The remaining menu options become enabled only after you select a request and view information.

    • Request Information displays information for the currently selected request and demographic information for its requestor. It allows you to either view or edit information, and to delete the request.

    • Patient Information displays a list of patients whose records are included in the request. It also allows you to add, change, or delete patients on the request, to view documents, and to add, change, or delete patient documents to include in the request.

    • Billing & Payment allows you to view or edit billing and shipping information for the current request, as well as generate pre-bill letters or invoices. It also allows you to release the requested documents.

    • Invoice History displays a list of invoices generated for the currently selected request and view invoice details.

    • Release History displays a list of all release events for the currently selected request. It also allows you to filter the list by patient name, user name, or release date.

    • Event History displays a list of all events associated with the currently selected request. It allows you to filter the list by event name, originator, or date.

    • Comments displays a list of all comments associated with the currently selected request. It allows you to filter the list by author or date.

    • Letters allows you to generate a letter (either a cover letter or other type) for the currently selected request.

     

  • Reports

    The Reports menu contains an option for each report the ROI Module can produce. Each option allows you to enter appropriate report criteria and generate the target report.

     

  • Administration

    The Administration menu contains the following options.

    • Billing Maintenance provides access to submenu options that allow administrators to define and maintain media types, billing tiers and fees, billing templates, requestor types, page weight, and delivery and payment methods.

    • Reasons Maintenance provides access to submenu options that enable administrators to define and maintain adjustment, request, and request status reasons.

    • Other Maintenance provides access to submenu options that allow administrators to define and maintain document types, letter templates, customized notes, and Social Security Number masking.

    • Configure Output Settings allows authorized users to configure output queues and to configure settings for print, fax, and file output methods, including header, footer, and watermark attributes. You must have a valid login to access configurable settings.

     

  • Help

    The Help menu contains these options:

    • ROI Help provides access to the ROI online help. The online help can also be launched by pressing the F1 key.

    • About ROI displays the release and version number for the ROI client.
