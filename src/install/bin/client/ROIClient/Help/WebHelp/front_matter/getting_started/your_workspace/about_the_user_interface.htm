About the user interface

User permissions

Your access to and use of the ROI Module is controlled by your MPF Workstation user profile. Your profile identifies the ROI functions you are permitted to use, and the ROI Module consequently displays information, options, and buttons supported by your permissions. If you cannot perform an operation, you most likely do not have appropriate permissions, and you should consult your system administrator.

Windows and dialog boxes

The McKesson Patient Folder ROI Module window is divided into sections. Each section offers specific types of information. The window also uses standard features such as a title bar, a menu bar, and tabs.

Certain actions on the module window launch dialog boxes in support of the current function. Dialog boxes are modal, meaning you cannot continue until you close the dialog box by specifically saving, canceling, or otherwise handling your dialog box entries.

Buttons, menus, and tabs

Buttons, menus and menu items, tabs, and other screen elements are visible or invisible and enabled or disabled depending on multiple factors, such as your security rights, the nature of your operation, and preconditions. Elements can be:

  • Enabled

    Some elements are perpetually enabled because they always meet preconditions. For example, options in Help are available to you at any time, without regard to contingencies or security levels, and you can elect to cancel an operation, jump between tabs or cascading menus, or exit the system at any time.

  • Disabled

    Some controls remain disabled until all logical preconditions for a certain action are met. For example:

    • Buttons that are not yet usable at a certain point in an operation are visible, but disabled. If, based on your actions, a button becomes relevant to the operation (for instance, if you enter all required information into a given screen), the button becomes enabled. You may then proceed. An example is the OK button on the Change Password dialog box, which is enabled only after you enter all required data.

  • Absent

    Some controls display only to certain users, based on security rights to perform associated actions. As a general rule, if a user will never have an opportunity under any circumstances to use a given element, it is not displayed on the interface.
