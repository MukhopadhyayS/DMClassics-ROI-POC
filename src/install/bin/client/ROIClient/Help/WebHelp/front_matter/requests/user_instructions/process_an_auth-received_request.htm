Process an Auth-Received request

This task tells you how to open and process an Auth-Received request.

  1. Find a request. Include either the "Auth-Received" or "Auth-Received Denied" request status in the search criteria.

  2. In the search results grid, double-click on the request you want to process. The system displays the Request Information form.

  3. As a best practice, click the View Authorization Request button to verify information for this request.

  4. If information is valid, continue with instructions below for the type of request you are processing. Otherwise, follow procedures specified by your site for handling invalid information on an authorization.

 

To process Auth-Received requests

  1. Enter information on the Patient Information form.

    1. Click Add Patient, add one or more patients to the request, and return to the Patient Information form.

    2. Select a patient you just added. At the document tree view, select the document(s) you want to release for this patient and click Add to add them to the Document(s) Selected for Release box.

    3. Repeat for the next patient, if one exists.

    4. Click Save & Bill to continue building the request.

  2. On the Billing & Payment Information form, enter billing information and shipping information.  

  3. If no pre-bill is required, click Release. (Otherwise, see Pre-bill a request for further instructions.)

Note: The Release, Pre-Bill, and Invoice buttons are disabled if unsaved changes exist. Either save or revert the changes to enable the buttons. The Re-Release button is enabled only if you selected previously released documents in the DSR document tree on the Patient Information form.

  1. At the Release dialog box, specify a cover letter template and an invoice template, and click Continue. When the system displays the Preview of Release Letter(s) dialog box, click Continue. After output is successfully generated, the system returns you to the Billing & Payment Information form.

  2. The release process is complete within the ROI system. Complete any manual steps (such as handing off to the shipper). You will be able to access this request later to record payments, to re-send or re-bill, or enter notes as needed.

 

 

To process Auth-Received Denied requests

The system automatically sets the Request Status to Denied.

Generate a letter to the requestor to explain the disposition of the request.
