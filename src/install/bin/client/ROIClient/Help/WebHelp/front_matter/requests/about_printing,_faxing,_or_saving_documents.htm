About printing, faxing, or saving documents

Patient documents stored in the MPF Workstation database are displayed in the WinDSS document viewer that is available with the base Patient Folder product, and you can use WinDSS view and print functions for those documents when needed. Any document rendered in the document viewer is also supported by the ROI output service. The ROI Module enables you to print, fax, or save to PDF all release documents, including letters, invoices, and patient documents.

The system outputs scanned documents at the same scale used during capture and not as rendered in the viewer (for example, a driver’s license is printed based on how it was scanned and not on best fit). The system shrinks images to fit the selected paper size while accommodating header, footer, and print margins. Margins are set by the document capture applications (that is, scan, COLD, or HL7). Color output depends on your print device; if color is not supported, the output is rendered in grayscale.  

Permissions

Your MPF Workstation security permissions control which output options you can use.

  • To print or fax, you must have the ROI Print/Fax permission.

  • To save output to a PDF file, you must have the ROI Export to PDF permission.

The ROI Module does not enforce your MPF Workstation facility-level print/fax security right when releasing documents or generating letters, invoices, pre-bills, and reports. However, your MPF Workstation facility-level security right is enforced when printing documents from the viewer (meaning you could have the right to print from the viewer at one facility, but not at another).

If you cannot perform an operation, you most likely do not have appropriate permissions, and you should consult your system administrator.

Patient identifiers in headers and footers

The output service supports printing headers, footers, and watermarks on each page of a released document, using application metadata specified by your system administrator.

For the ROI Module, these features are enabled in the ROI Administration function by an authorized user and apply to all released documents. When headers and footers are enabled, the configured patient identifiers are printed on each document page that is released. They are not included on invoices, cover letters, certification letters, other letters, or page separators.

The following patient identifiers are applied to released document pages: Patient Name, Patient MRN, Patient Encounter, and EPN (for sites that are EPN-enabled.

Output options

You can elect to print or fax output, or save output to a PDF file. You may also modify output properties that are governed by the output option you choose, such as fax numbers or number of copies to print.

Printed or faxed documents are output in the following order: cover letter (if selected), invoice letter (if selected), and the documents that were selected for release.

PDF output is useful when you need to ship large volumes of patient documents to a requestor. The PDF files can be transferred to a CD or other media for shipment, bypassing costly paper output.

What does the PDF output include?

For PDF output, each release may consist of multiple PDF files. The release includes one PDF file for each unique request ID-facility-MRN combination that exists on the request and for which documents are being released. If you generate letters and/or invoices for the release, the release also includes a supplementary PDF file that contains the letters and invoice you generated. Requestors can associate supplementary PDF files with the patient PDF files by virtue of the request ID in the file names.

How are the PDF files named?

The system automatically names the PDF files.

  • The supplementary filename format is:  <request-ID>-Supplementary-v<rel-sequence-num>.pdf.

  • The naming convention for patient document files is:

<request-ID>-<facility>-<MRN>-<patient-name>-v<rel-sequence-num>.pdf

where:

request-ID

=

An ID generated by the ROI system.

<facility>-<MRN>

=

The facility code and MRN associated with the patient.

patient-name

=

LastName-FirstName format.

rel-sequence-num

=

A sequence number assigned by the system, such as 1, 2, 3, and so on; the sequence number is incremented by 1 each time this request has a release. Required.

Note: Very rarely, a patient document PDF file may exceed the permitted maximum file size, causing the system to break the single PDF file into parts. To identify each part, the system precedes the sequence number with a part number (001, 002, and so on). For example:

10011-A-123456-Smith-John-001-v1.pdf
10011-A-123456-Smith-John-002-v1.pdf

When combined in the indicated sequence, the "parts" form a single document.

More examples of file names used for release documents are shown below:

10011-Supplementary-v1.pdf
10011-A-123456-Smith-John-v1.pdf
10011-B-654321-Smith-John-v1.pdf

 

Where do the PDF files reside?

You cannot choose the save location for the PDF files. The system automatically creates the files in a shared directory path configured by your ROI system administrator. The path includes your MPF Workstation user name as the name of the directory where your output files will reside.

Path format is:

\\<server-name>\<shared-dir$>\<your-username-dir>\

Normally, ROI users do not need to access PDF files once they are created. The files exist only for downstream use by staff personnel who are responsible for shipping the files on physical media.

 

Output queues

The system remembers across application sessions the last queue you used for each type of output and uses it as the default output queue for the next output request of the same type.
