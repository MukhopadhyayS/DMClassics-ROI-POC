Find a request

This task enables you to search for and select a specific request. The Find function supports a wide variety of search criteria that can be used in any combination, enabling you to narrow results to a concise list of requests.

For example, you can search by:

  • Request status (to list all requests with a specific status)
  • Patient identifiers (to list all requests for a specific patient, date of birth, SSN, encounter number, MRN, or EPN)
  • Requestor identifiers (to list all requests for a specific requestor or requestor type)
  • Balance amounts (to list all requests with a balance that is less than, equal to, or more than a specific dollar amount)
  • Facility, received date, invoice number, request ID, or completed date

Note: You can access requests that include patients with VIP and/or locked items only if you have the ROI VIP status and/or Access Locked Records permissions. If you do not have permissions, the system alerts you to the existence of VIP and/or locked records.

  1. Select Requests > Find / Create Request to access the Find / Create Request form.
  1. In the main content pane, enter search criteria.

You can use any search criteria in any combination, with the following exceptions.

- If you use Last Name and/or First Name, you must enter at least three characters in one or the other, or across both fields.
- If you use Date of Birth, you must combine it with at least one other field.
- If you use Facility, you must combine it with at least one other field.
- For SSN, Enc/Acct, MRN, or EPN, you must enter at least three characters.
  1. Click Find Request.  The system lists all requests that meet the search criteria you entered.
- The result set is limited to maximum of 500 records. If more records exist, you can enter more search criteria to narrow the search. In addition, the Patient(s) column in the detail pane lists up to 20 patients at a time for each request. If more patient names exist on a request, the system appends an ellipsis (...) to the list of names. To display the remaining names in the list, hover the cursor over that cell.
- If a request contains records from facilities that you are not allowed to access, the detail pane lists the request in red text and masks data in the following columns: Requestor Name, Requestor Type, Patient(s), and Subtitle. If you select the request, the system disables the View/Edit Request button.
  1. In the detail pane, select the request you want and click View/Edit Request. (You cannot view or edit a restricted request.) The system displays the Request Information form.

If you selected an Auth-Received or Auth-Received Denied request, the system changes its status to Logged. If you want to process this request, as a best practice click View Authorization Request to verify information for the request. Then proceed with other tasks for processing the request, such as adding documents, generating letters, shipping, billing, and so on.
