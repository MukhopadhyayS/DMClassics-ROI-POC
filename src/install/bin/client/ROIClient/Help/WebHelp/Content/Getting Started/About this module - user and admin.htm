About this module

The OneContent Release of Information (ROI) Module provides a tool for managing all requests for copies of medical records, including a billing component. The ROI Module enables you to select and release patient information to entities who request patient records, and to easily generate letters, billing notices, and invoices for that service. You can manage and account for the release of both OneContent and non-OneContent records.

Release of Information features enable users to:

• Configure and manage administrative functions including user profile permissions and perform maintenance.
• Create and manage patients and view patient information, records and request history.
• Create requestors and view their profile and request and account histories.
• Manage requestor accounts including adding payments and adjustments, generating statements, and issuing refunds.
• Create requests specifying billing and shipping information, output method, and sales tax.
• Generate cover letters, statements, and invoices (both current and past due).
• Generate reports.
• Find overdue invoices.

The following items apply globally for ROI administrators:

• Any changes you make apply going forward; they are not retroactive.
• Any options you change apply across the enterprise; they are not facility specific.
• Data you enter (such as names and descriptions for items) is not case sensitive.
• Users who are logged in when you make changes must log out and then log back in for the changes to take effect.
• The data on many ROI forms is in grid format. You can sort the data in alphabetical order or reverse-alphabetical order by clicking the column header.

Note: With the exception of configuring output settings, all administration features can be accessed from the left pane of the Administration tab. All features, including configuring output settings, can be accessed from the Administration menu bar.
