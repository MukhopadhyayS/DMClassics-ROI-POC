Patient Information form

The Patient Information form enables you to perform a wide variety of tasks related to a currently selected request. You can:

  • View a list of documents for the currently selected patient
  • View or edit the patient profile
  • Add another patient to the request
  • Add attachments to a patient's record
  • Select documents and attachments to release
  • Create or edit non-OneContent documents
  • Create attachments and edit attachment information
  • Filter the document list with various criteria
  • Generate billing for a request

Note: You can access this form at any time while creating a new request or viewing an existing request.

To open the form

  1. Select the Requests tab.
  2. Search for and select a request.
  3. Click View/Edit Request.
  4. Select the Patient Information function.

Item descriptions

Requestor information box

Identifies the requestor type, the requestor's name and demographic information, and the requirements for pre-payment and certification letters (if any). Also displays the request ID, current status, and receipt date and time.

Patient(s) Selected for This Request group box

Patient(s) on this Request

Lists all patients for whom records are included in this request, in alphanumeric ascending order, and allows you to select a single entry. Your selection drives the information that displays adjacent to the list box.

Read-only.

Other rules are:

  • If you selected to create a new request from the Patients or Requestors tab:
  • Any patients you selected on the Patients tab are automatically added to the request.
  • For a Patient Requestor, any patients you selected from the Create New Request with Patient Requestor dialog box are automatically added to the request.
  • If you clicked Patient Requesting from the Patients tab, the selected patients are automatically added on this request
  • For EPN systems, the same patient is listed only once. For non-EPN systems, the same patient may be listed multiple times. If multiple patients on the list have the same name, the system uses the patient identifier (EPN or Facility (first) /MRN (second)) to sort patients in alphanumeric ascending order.

Patient information fields

Displays information for the patient currently selected in the adjacent list box. Read-only. Information includes:

  • Name
    For a new request with no patients added, displays None Selected.
    For an existing request or if at least one patient has been added, displays the selected patient’s name.
    Displays the   icon when the patient is a VIP or has a VIP encounter.
    Displays the   icon when the patient is Locked or has a Locked Encounter.
  • Gender
    Blank for a new request with no patients, or if it is missing from an existing request. Otherwise, displays the patient's gender.
  • DOB
    Blank for a new request with no patients, or if it is missing from an existing request. Otherwise, displays the patient's date of birth.
  • SSN
    Blank for a new request with no patients, or if it is missing from an existing request. Otherwise, displays the patient's SSN.
  • Facility
    Blank for a new request with no patients, or if it is missing from an existing request. Otherwise, displays the facility where patient care occurred.
  • MRN
    Blank for a new request with no patients, or if it is missing from an existing request. Otherwise, displays the patient's MRN.
  • EPN
    Displays only for EPN systems. Blank for a new request with no patients, or if it is missing from an existing request. Otherwise, displays the patient's EPN.

Add Patient

Allows an authorized user to add a patient to the request.

Displays only for a new request.

Launches the Add Patient dialog box.

View / Edit Patient Profile

Allows you to view patient information, and enables an authorized user to edit information.

Displays only for an existing request.

Launches the View / Edit Patient's Profile dialog box.

Remove Selected Patient

Allows an authorized user to delete a patient from the request.

Displays only for an existing request or after at least one patient exists on a new request.

Disabled if you select a patient who has at least one document released.

Does not prompt for confirmation to delete.

Add Another Patient

Allows an authorized user to add a patient to the request.

Displays only for an existing request or after you have added at least one patient to a new request.

Launches the Add Patient dialog box.

Record View

Allows you to filter documents by selecting a record view.

Lists all record views that you have permission to access.

Enabled only for OneContent non-global documents.

Defaults to the first record view option in the list.

Sort By options

Sorts documents according to the option you choose. Options are Encounter and Document.

Enabled only for OneContent patients and documents

Has no affect on OneContent global documents or non-OneContent documents

Defaults to Encounter

  • Displays documents by encounter in chart order
  • Sort order is by Encounter (most recent first), then by document type (in Chart order), then by version, most recent version first (if view versions is enabled), then by page number (ascending)

When Document is selected:

  • Displays documents alphabetically by Document Name by encounter
  • Sort order is by document type (alphanumeric ascending), then by encounter (most recent first), then by version, most recent version first (if view versions is enabled), then by page number (ascending)

Only Disclosure Docs option

Filters records to show only disclosure documents. The disclosure document is a form completed by the patient to identify one or more entities from which to withhold all or part of the patient's records. The facility cannot release the patient's documents to any entity listed on the form.

  • Displays only if you have permission to view disclosure documents
  • Enabled only for OneContent patients and documents
  • Defaults to off
  • When checked, the records tree control shows only documents marked as disclosure documents

Show Versions

Shows document versions in the patient record view.

Displays only if you have permission to view document versions.

Enabled only for OneContent patients and documents

Defaults to off, and the records tree control shows only the page numbers for the most recent version.

When checked, the records tree control shows all versions of all documents in the record view.

Document tree

Allows you to navigate to and select records associated with the current patient.

For OneContent patients, displays Global, OneContent, non-OneContent, and attachment documents. For non-OneContent patients, displays only non-OneContent and attachment documents.

  • The "GLOBAL DOCUMENTS" folder lists all documents for the selected patient that are classified in OneContent as global. Display is in chart order, most recent first.  Information includes document type name and admit date and time. Display is not affected by Filter, Record View, or Sort By selections.
  • If non-OneContent documents exist for this patient, the tree displays a "NON MPF DOCUMENTS" folder. The node lists documents, most recent first, with document type name, facility, department, date of service, and pages. Display is not affected by Filter or Sort By selections.
  • If attachments exist for this patient, the tree displays an "ATTACHMENTS" folder. The node lists documents that have been attached to the patient's record.
  • Remaining folders represent OneContent documents and are sorted according to the Sort By option. OneContent documents are grouped by Encounter, Document Names, Version, and Page numbers. At the encounter level, the system displays encounter number, patient status, facility and admit date.

Also identifies (for OneContent documents only):

  • Encounters with one or more deficiencies () and, if you have adequate permissions, identifies VIP patients () and locked encounters ().  
  • If the primary plan for this encounter is self-pay.

> (Add)

Adds the selected item to the list of document(s) selected for release.

Enabled only when a request has at least one patient added.

Does not add duplicate records.

>> (Add All)

Adds all selected items to the list of document(s) selected for release.

Enabled only when a request has at least one patient added.

Does not add duplicate records.

Enc Details button

Displays encounter information for the encounters currently selected in the document tree.

Enabled only when one or more OneContent encounters are selected.

Launches the Encounter Details dialog box.

View Selected button

Opens currently selected global and/or a OneContent document in the document viewer. Non-OneContent documents cannot display in the viewer.

Displays only for OneContent patients (not available if you have accessed a non-OneContent patient).

Enabled only after you select at least one Global or OneContent encounter or document.

Disabled if only Non-OneContent documents are selected.

View All button

Opens all global and/or OneContent documents in the document viewer. Non-OneContent documents cannot display in the viewer.

Displays only for OneContent patients (not available if you have accessed a non-OneContent patient).

Enabled only after you select at least one Global or encounter or document.

Disabled if you select the Global Documents or Non-OneContent folder or a non-OneContent document.

New Document button

Enables you to create a new non-OneContent document for the current patient.

Launches the Create New Non-MPF Document dialog box.

Edit Document button

Allows you to edit a selected non-OneContent document for the current patient.

Enabled only after you select at least one non-OneContent document.

Launches the Edit Non-MPF Document dialog box.

Add Attachment

Allows an authorized user to add an attachment to the patient record.

Launches the Create New Attachment dialog box.

Edit Attachment

Allows an authorized user to edit information for an attachment or delete/withhold an attachment from the patient's record.

Launches the Edit Attachment dialog box.

Documents(s) Selected for Release group box

Filter by options

Allows you to filter the release document list. Options are:

  • All - displays all documents that have been selected to release
  • Released - displays only those documents that have been previously released
  • Unreleased - displays only those documents that have been selected to release but have not yet been released

Enabled only after at least one document exists in the release list.

Release documents tree view

Lists all documents you have selected for release on this request. Any previously released document is highlighted in green.

Any document that has been moved/deleted from the patient is highlighted in red and must not be selected. These documents may/may not have been previously released. If the document has not been released, you can remove the document from the tree.  However, previously released documents cannot be moved from the tree.

By default, all previously released documents are unchecked and all unreleased documents are checked.

If a parent check box is checked, all subordinate check boxes are automatically checked. If you deselect some of the subordinate items, the parent check box changes to solid to indicate mixed use.

Remove

Removes eligible selected items from the list of release documents. Previously released documents cannot be removed.

Enabled when the request has at least one document on the release document list.

Disabled when:

  • The request is new and no patients have been added
  • No patients exist
  • No documents have been checked
  • You select a previously released document or when the Filter by option is set to Released

Remove All

Removes eligible selected items from the list of release documents. Previously released documents cannot be removed.

Enabled when the request has at least one document on the release document list and at least one unreleased document.

Disabled when:

  • The request is new and no patients have been added
  • No patients exist
  • You select a previously released document, when the Filter by option is set to Released, or when the Filter by option is set to All and previously released documents are displayed

Modify Billing Tier

Allows you to override the current billing tier for a selected non-OneContent document.

Enabled only after you select a non-OneContent document in the Document(s) Selected for Release box.

Launches the Modify Billing Tier dialog box, where you can select from a list of defined billing tiers.

Resend

Allows you to resend previously released documents without generating billing charges, Accounting of Disclosure or release history entries, or page count updates.

Enabled when only previously released documents are selected in the Documents Selected for Release box.  Previously released documents must not be deleted documents.

Launches the output dialog box depending on the shipping method selected during the last release or re-release event. The system returns you to the Patient Information screen after the output operation is complete, clears all selections, and disables this button.

View Authorization Request

Allows you to view the authorization request that initiates an ROI request.

Enabled only when the document type has been enabled to support authorization requests.

Save

Saves current request information. Effective for both new and existing requests.

Enabled only after all required fields contain data or a change is made to existing information, and you have selected at least one document (black or green, but not both and not in red) to release or resend.

Save & Bill

Saves current request information and proceeds to the next step for completing a request. Effective for both new and existing requests.

Enabled only when:

  • You have added a document to an existing request or
  • You have created a new request with at least one patient and one document or
  • You have selected at least one document (black or green, but not both and not in red) on the request to release or resend

Launches the Billing & Payment form.

Revert

Reverts all information on the Patient Information form to the last saved state.

Displays only for an existing request.

Enabled only after you edit information.

Cancel

Displays only for a new request.

Cancels the create request action and returns you to the calling form.
