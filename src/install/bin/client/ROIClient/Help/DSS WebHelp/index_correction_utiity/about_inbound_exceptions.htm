About inbound exceptions

What triggers an inbound exception?

An inbound exception occurs when any of the following conditions exist:

  • The system already has a locked version of the incoming document. If the system is configured to support locked documents, documents are locked only after they have been legally authenticated.
  • The incoming document is already authenticated or legally authenticated in the system.
  • A copy of the incoming document exists in the system and has been assigned deficiencies.
  • An archived copy of the incoming document exists in the system.
  • A copy of the incoming document exists in the system and is currently checked-out by a physician for deficiency completion.
  • The incoming document has a status of legally authenticated, but the document does not exist in the system.
  • The incoming document is a COLD document and is authenticated, but does not exist in the system.

How exceptions are handled

For each inbound exception condition, customers can specify whether to auto-ignore the inbound document, auto-accept the inbound document, or flag the inbound document for manual review. Documents that require manual review are written to the ICU inbound exception queue.

Manual review permits you to determine whether the inbound document should be retained or discarded. You review inbound exceptions by manually comparing an incoming document to an existing document with the same index information, or by otherwise evaluating the exception to determine disposition.

When to review inbound exceptions

Review inbound exceptions at any time or according to procedures established by your facility.
