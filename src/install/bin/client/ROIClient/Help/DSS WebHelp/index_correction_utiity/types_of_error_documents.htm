Types of error documents

The Index Correction Utility displays information for these types of pending error documents:

  • Non-expired documents. Non-expired documents are those that still have upload attempts remaining. The Index Correction Utility cannot change or process non-expired documents.
  • Expired documents. An expired document has failed the preset number of upload attempts and requires manual correction.
  • Inbound exceptions. These are incoming documents that encounter specific conditions within the system. Most of the time, an exception is triggered because the document already exists in the system. Your System Administrator configures how the system handles inbound exception events. Some can be handled automatically, but others need manual intervention. During manual processing, you review the reason for the exception and potentially compare an existing document to the incoming document to determine which document to keep and which document to discard. See “About inbound exceptions” below for more information.

Documents that fail all upload attempts ultimately require user intervention to troubleshoot the failure. The document retains index information from the Index Upload file to assist in troubleshooting the errors.
