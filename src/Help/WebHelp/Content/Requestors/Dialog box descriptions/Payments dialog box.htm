Payments dialog box

The Payments dialog box allows you to manage payments for a requestor's account.

To open the form

  1. Find a requestor.
  2. Select Requestors > Account Management.
  3. In the Account Management form, click Add Payment. The system displays the Payments dialog box.

Item descriptions

Amount

The amount of the payment

Method

The method of the payment, for example, check or credit card.

Note

A note describing the payment.

Date

The date of the payment.  The default is today's date. Click the calendar icon to select a different date.

Payment Amount

The payment amount entered in the Amount field.

Applied

The amount to be applied.

Refunded

Any amount refunded.

Note: You cannot apply a refund in this dialog box.  See Issue a refund.

Unapplied

The amount from the entered payment that has not been applied.

Restricted facility permissions (conditional)

A note indicating that other invoices may exist and those invoices are displayed based upon facility permissions.

Apply payment to

  • Invoice/Pre-Bill #
    List of all open invoices and pre-billings. Sorted chronologically with the oldest first.
  • Request ID
    Request ID associated with the invoice.
  • Date
    Date the invoice was created.
  • Charges
    Original charges when invoice was generated.
  • Pay/Adj
    Total payments/adjustments for the invoice.
  • Applied
    Amount of the payment previously applied to the invoice.
  • Balance
    Current invoice balance (charges - payments + adjustments).
  • Apply
    Manual override of amount of the total payment to apply to individual invoice. Also displays the amount to apply

Save

Saves the new or modified payment.

Cancel

Cancels the payment you were adding or any unsaved changes made to an existing payment.

Remove Payment (conditional display)

Removes the selected payment.  

Enabled only when an unapplied payment is selected on the requestor's account register.
