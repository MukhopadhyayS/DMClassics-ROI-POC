Configure Default Unbillable Request form

The Configure Default Unbillable Request function allows you to specify whether requests should be released as billable or unbillable events. You can override the default setting during the release and invoice process. You must be an ROI Administrator to configure this option.

To open the form

Select Administration > Other Maintenance > Configure Default Unbillable Request.

Item descriptions

Billable

Indicates whether to make released requests billable.

Unbillable

Indicates whether to make released requests unbillable. This is the default.

Save

Saves the setting.

Cancel

Discards any unsaved changes.
