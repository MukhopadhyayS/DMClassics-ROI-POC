Other Maintenance

About Other Maintenance

The Other Maintenance functions provide ROI administrators the capabilities described below.

This functionality is accessed by selecting Other Maintenance on the left pane of the Administration tab.

Designate document types

For ROI purposes, administrators can flag any document type as being:

  • A disclosure document. This flag enables the ROI Module to filter the document tree to display only disclosure documents.
  • An authorization request.  Currently not implemented. Reserved for future use.
  • A Meaningful Use document.  This setting enables you to flag a document type as a Clinical Summary, a Discharge Summary, or as Discharge Instructions. These are collectively referred to as Meaningful Use documents. The ROI Module calculates turnaround times for any Meaningful Use document.
Maintain letter templates

The ROI Module provides predefined letter templates that can be used for generating letters and invoices related to ROI requests. Administrators can download, modify, and upload the templates to meet the needs of the organization. They can also create new templates using Microsoft Word.

Letter templates contain valid data pointers that save you time by automatically inserting data from the system database into the letter.

Configure notes

Administrators can configure standard notes that users can add to letters and invoices associated with ROI requests.

Mask Social Security Numbers

Administrators can set the option to mask Social Security Numbers so they do not display during online ROI sessions or in reports, letters, and invoices generated by ROI users.

Configure clinical systems

External clinical systems (like the Paragon product) generate original documents that sometimes need to be acquired by ROI Module users and attached to an ROI request. The module must be able to connect to the external system, verify patient identifiers, and perform the download. In addition, to show compliance with legal regulations, the clinical system may require follow-up notifications, such as when such documents are actually released on a request. Consequently, ROI administrators have the ability to maintain settings that are associated with the clinical systems and can even add a new clinical system instance to the ROI Module.

Configure location

Administrators can configure the default country location value as part of address information. Users can override this default.

Configure a default setting for unbillable requests

Administrators can configure a billable or unbillable setting that applies to all requests until you change it. The setting is an enterprise setting and is in effect for all system users. By default, the setting is configured as unbillable.
