Adjustment Reasons form

The Adjustment Reasons form allows you to view, add, modify, or delete the reasons users can select when adjusting the balance due for an ROI request. Examples of adjustment reasons include Order Change, Billing Mistake, and Additional Charge.

To open the form

Select the Administration tab > Reasons Maintenance > Adjustment Reasons.

Item descriptions

Main Content Pane

Adjustment Reasons grid

Displays the adjustment reasons and associated information currently defined in the system. The grid title is preceded by the number of existing adjustment reasons.

The Adjustment Reasons grid contains the following columns:

  • Reason Name
    Names of the adjustment reasons currently defined in the system.
  • Display Text
    Display text associated with the adjustment reasons. This text displays on letters if that attribute has been assigned to the letter template, in the audit trail report if configured for this, and in the system-generated history log.

Create New

Allows you to enter the information needed to add a new adjustment reason.

Delete

Deletes the selected adjustment reason. You can delete adjustment reasons at any time.

Detail Pane

Reason Name

Specifies the name for the adjustment reason.

Adjustment reason names must be unique. They are not case sensitive. You can change them at any time.

Display Text

Specifies the text to display for the adjustment reason. This text displays on letters if the attribute has been assigned to the letter template, in the audit trail report if configured for this, and in the system-generated history log.

Display text must be unique. It is not case sensitive. You can change it at any time.

Save

Saves the new or modified adjustment reason.

Cancel

Cancels the adjustment reason you were adding or any unsaved changes you made to an existing adjustment reason.
