Request Status Reasons form

The Request Status Reasons form allows you to view, add, modify, or delete the reasons that users can select when adding or changing the status of an ROI request. Examples of request status reasons include Revocation and Wrong Patient.

For information about request statuses, see About requests.

To open the form

Select the Administration tab > Reasons Maintenance > Request Status Reasons.

Item descriptions

Main Content Pane

Request Status Reasons grid

Displays the request status reasons and associated information currently defined in the system. The grid title is preceded by the number of existing request status reasons.

The Request Status Reasons grid contains the following columns:

  • Request Status
    Request statuses associated with the request status reasons currently defined in the system.
  • Reason Name
    Names of the reasons associated with the request statuses.
  • Display Text
    Display text associated with the request status reasons. This text displays on letters if the attribute has been assigned to the letter template, in the audit trail report if configured for this, and in the system-generated history log.

Create New

Allows you to enter the information needed to add a new request status reason.

Delete

Deletes the selected request status reason. You can delete the reasons at any time.

Note: You are deleting the reason, not the request status.

Detail Pane

Request Status

Specifies the request status for which you want to define a reason. The pre-defined request statuses are Auth-Received, Auth-Received Denied, Cancelled, Completed, Denied, Logged, Pended, and Pre-Billed.

Reason Name

Specifies the name for the request status reason. Request status reason names must be unique. They are not case sensitive.

You can change the name for a request status reason at any time.

Display Text

Specifies the text to display for the request status reason. This text displays on letters if the attribute has been assigned to the letter template, in the audit trail report if configured for this, and in the system-generated history log.

The display text must be unique. It is not case sensitive. You can change it at any time.

Save

Saves the new or modified request status reason.

Cancel

Cancels the request status reason you were adding or any unsaved changes you made to an existing request status reason.
