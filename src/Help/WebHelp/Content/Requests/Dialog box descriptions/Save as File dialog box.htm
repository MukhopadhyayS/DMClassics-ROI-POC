Save as File dialog box

The Save as File dialog box enables you to save to a single file any cover letters, invoices, pre-bills, statements and patient documents that are included in a release request.

You cannot choose the save location for the file. The system automatically creates the file in a shared directory path configured by the ROI system administrator; the path also uses your OneContent user name for the directory where your output files will reside.

Path format is: \\<server-name>\<common-dir$>\<ROI-PDF-dir>\<your-username-dir>\.

The actual filename includes <request ID><release sequence><part number><version number>. (See About PDF documents for more details.)

To open the form

The system launches the dialog box when you do any of the following:

  • Click Continue from the Release Documents dialog box without a letter selected
  • Click PDF from the Letters preview dialog box
  • Click PDF from the Invoice preview dialog box
  • Click PDF from the Pre-Bill preview dialog box

Item descriptions

PDF group box

Allows you to specify the intended media type, password, and name of a file to generate. Also allows you set file properties.

Note: Currently, only the PDF file type is supported.

  • For Media
    Displays a list of media types and allows you to select one.
    Options are CD, DVD, and Other / Unlimited Size.
    Your choice determines the maximum allowable file size in Mbytes, and the system creates multiple files, if needed. The maximum file size for DVD is 1GB.
    Defaults to Other / Unlimited Size.
  • Password
    Allows you to enter a password (up to 64 characters) to encrypt the PDF file.
    If flagged as required, then defaults to the password specified for the queue (if any), and can be overridden but cannot be left blank. If not required, then can be blank or set to a value of your choice. Must manually be given to the requestor to use when opening the file.
  • Name
    Lists all save-as-file queues that have been configured, in alphanumeric order.
    Defaults to the first save-as-file queue in the list for your first-time save-as-file job or when the last-used save-as-file queue has been deleted; otherwise defaults to the last save-as-file queue you selected.
  • Status, Type, Where, and Comment
    Read-only.
    Displays information for the queue you selected, including the queue status, the queue type, the destination where the output will go, and any comment associated with the queue.

Add to Released Document group box

Allows you to print a header, footer, and/or watermark on all released documents (not on letters).

Displays only when the output dialog box has been launched from the Release dialog box and the header, footer, and/or watermark options are enabled in the Configure Output Settings function (any disabled option does not display). Displayed options are enabled only if a corresponding value has been configured.

  • Headers and footers can display an absolute page number, a relative page number, the patient name, and the encounter number, MRN, or EPN.
  • Watermarks can contain either the default value or any text string you specify up to 20 characters.

OK

Writes the output to the selected file location as a PDF file and closes the dialog box.

If an error occurs, the system alerts you.

Cancel

Discards the job, closes the dialog box, and returns you to the calling form or dialog box.
