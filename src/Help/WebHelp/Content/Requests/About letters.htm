About letters

The term letters broadly refers to ROI cover letters of various types, invoices, and billing notices. All letters are based on templates that reside within the ROI system. You can generate letters regardless of the status of a request, and letters and invoices can be printed, faxed, emailed or exported to a PDF file.

Permissions

The OneContent ROI Module makes it easier for authorized users to generate letters when fulfilling ROI requests. Your ability to generate and output letters is controlled by the security settings for your OneContent user profile. If you do not have certain permissions, some buttons and menu options are hidden. If you cannot perform an operation, you most likely do not have appropriate permissions, and you should consult your system administrator.

Types of letters

The ROI Module categorizes letters into these types:

  • Cover Letter. Cover letters are generally used when you release or resend documents. The ROI Module is delivered with standard Certification Letter and Cover Letter templates.
  • Invoice. Invoice letters are generated when you are ready to send an invoice to the requestor for ROI charges. This usually occurs when you release or re-release documents. The module is delivered with one predefined invoice template.
  • Pre-Bill. Pre-Bill letters are generated when you must bill the requestor before fulfilling the ROI request. All pre-bill letter templates are listed in the Pre-Bill dialog box. The module is delivered with one predefined pre-bill template.
  • Other. This letter type is used as a catch-all category. The module is delivered with templates for a Pend letter, a Denial letter, Cancellation letter and Refund letter in this category.
  • Requestor Statement. Statements are generated for a requestor providing activity about the requestor’s account.  This can be output with the documents selected for release.  

You cannot add, change, or delete letter types.

Letter templates

Each letter type can have many associated letter templates. Letter templates contain valid data pointers that save you time by automatically inserting data from the system database into the letter. They also contain boilerplate or customized verbiage used by your enterprise and can optionally support a Notes feature.

The module is delivered with a number of standard templates.

Note: ROI administrators can create additional templates based on the standard templates. See Maintain letter templates for more information.

Notes for letters

ROI administrators can create predefined notes to include on letters. These predefined notes display only when you choose a letter template that supports the Notes feature. You can select one or more predefined notes to include on the letter or create your own free-form verbiage to include on the letter.

Generating letters

You can generate letters from either the Release & Invoice function or the Letters function on the Requests tab. The system displays the generated letter in a preview window and allows you to save it to a PDF file or to print, fax or email it. If you are releasing the requested documents, the preview window also provides a Continue function that allows you to proceed to the next step for releasing documents.

Generating letters is an audited event if audit event code "2" is enabled in the OneContent application in Facility Audit Trail Setup.
