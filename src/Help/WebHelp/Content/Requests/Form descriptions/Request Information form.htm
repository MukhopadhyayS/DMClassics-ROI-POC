Request Information form

The Request Information form enables you to view and/or change information for the currently selected request. You can add or remove patient records to or from the request, view or edit the patient profile, select which records to release, add or edit non-OneContent documents, and either save the request for later processing or save it and bill it at the same time.

Note: You can access this form at any time while creating a new request or viewing an existing request.

To open the form

  1. Select the Requests tab.
  2. Do either of the following:
- Click New Request (to create a request).
- Search for and select a request, then click Open / View Request (to edit an existing request).

Item descriptions

Note: Date and time format are controlled by the regional settings of your operating system.

Request information fields

Request ID label

A unique ID assigned by the system to the request.

Read-only.

Request Reason box

The reason for this request. Allows you to select from reasons established by your system administrator.

  • For a new request, defaults to Please Select.
  • For an existing request, displays the current request reason as read-only text. If the request has a status is Complete or Cancelled, displays as read-only text; if the status is changed, you can select a different reason. Otherwise, allows you to select a different reason if needed.

Receipt Date box

Datetime stamp that the ROI department receives a request.

Allows you to select a date.

  • For a new request, defaults to blank until you click the drop-down arrow. Then displays today's date and the calendar date-picker tool. Includes the time stamp as well.
  • For an existing request, displays the datetime that the document was released into the system.

Date Created label

Date the system creates an ROI request.

Read-only.

Displays a value after a new request is saved.

Last Updated label

Date the request was last updated by a user or system actor.

Read-only.

Displays a value after a new request is saved.

Updated By label

The full name of the user or system actor who last updated the request.

Read-only.

Displays a value after a new request is saved.

 

Request Status group box

Current Status

For existing requests, displays the current status of the request.

For new requests, is blank until after the request is saved, and then defaults to Auth-Received (or Auth-Received Denied) if the system created the request or to Logged if a user created the request.

Can be changed by an authorized user.

Current Reason

The reason(s) associated with a status change (if the status requires a reason to be given).

Displays the latest recorded reason(s) from the Change of Status and the Status Reason Added events in the event history.

Can be changed by an authorized user.

Update Request Status button

Launches the Update Request Status dialog box so that an authorized user can change the request status and/or status reason(s).

Select Requestor or View/Edit Requestor's Profile button

Allows an authorized user to choose a requestor for this request or to view and/or edit requestor information.  

  • For a new request, the button is labeled Select Requestor and launches the Select Requestor dialog box.
  • For an existing request, the button is labeled View/Edit Requestor's Profile and launches the View / Edit Requestor's Profile dialog box.

Disabled if the Request Reason is "Migration."

Change Requestor button

Displays only for an existing request.

Allows an authorized user to choose a requestor for this request.

Launches the Change Requestor for Request dialog box.

Disabled if the Request Reason is "Migration."

Requestor Type

The type associated with the specified requestor. Blank if the request status is Auth-Received and for new requests.

If requestor type is Patient, displays the gender, date of birth, and Social Security Number for the patient requestor.

Read-only.

Contact Information group box

Displays requestor contact information (names and numbers) associated with this request.

For a new request, all fields become enabled after you select a requestor.

For an existing request, an authorized user can edit information in any field.

 

Buttons

View Authorization Request

Allows you to view the authorization request that triggered the ROI request.

Enabled only when the document type at the OneContent has been enabled to support authorization requests.

Save or Save & Continue

Saves the request information and proceeds to the next step in completing this request.

For a new request, button label is Save & Continue. Enabled only after all required fields are complete.

For an existing request, button label is Save. Enabled only after you edit information.

Revert

Reverts all information on the Request Information form to the last saved state.

Displays only for an existing request.

Enabled only after you edit information.

Cancel

Displays only for a new request.

Cancels the create request action and returns you to the calling form.

Delete Request

Displays and is enabled only if you have permission to delete requests.

Allows you to delete the request. Launches a confirmation prompt.

Disabled when one or more documents on the request have been released.
