About PDF documents

PDF output is useful when you need to ship large volumes of patient documents to a requestor. The system generates one PDF file for all OneContent documents, one for all non-OneContent documents, and one for all attachments that are selected for output to PDF. The PDF files can be transferred to a CD or other media for shipment, bypassing costly paper output.

Passwords for PDF documents

Two types of passwords may exist for PDF documents to provide security against unauthorized access:

  • A queue password, which is optional. Released documents are password protected only if a queue password exists.

Administrators specify whether to require a queue password while configuring a "Save as file" queue and can optionally specify a default password. Even if a password isn't required, you can still create a queue password when you generate the release documents. A queue password exists either by default or because you apply one as you generate the PDF documents for release.

  • A request password, which is always automatically generated only once and applies to all documents for a specific request.

The request password applies to the original release of documents for a request as well as for subsequent releases for that request (for example, if you add documents to the request, resend documents, or re-release documents). It is effective only when a queue password exists.  

Requestors can use either the queue or request password to open password-protected documents. The queue password gives the user administrator rights to the PDF; the request password gives the user limited rights to the PDF.  Follow procedures established by your site to convey the password(s) to the requestor in a secure manner.

Passwords are displayed in clear text on these windows only when both passwords exist:

  • Requests > Invoice History
  • Requests > Release History
Additional details

How are the PDF files named?

The system automatically names the supplementary and patient document PDF files.

  • The supplementary filename format is:  <request-ID>-Supplementary-v<rel-sequence-num>.pdf.
  • The patient document filename format is:

<request-ID>-<facility>-<MRN>-<patient-name>-v<rel-sequence-num>.pdf

where:

request-ID

=

An ID generated by the ROI system.

<facility>-<MRN>

=

The facility code and MRN associated with the patient.

patient-name

=

LastName-FirstName format.

rel-sequence-num

=

A release sequence number assigned by the system, such as 1, 2, 3, and so on; the sequence number is incremented by 1 each time this request has a release.

Note: Very rarely, a patient document PDF file may exceed the permitted maximum file size, causing the system to break the single PDF file into parts. To identify each part, the system precedes the release sequence number with a part number (001, 002, and so on). For example:

10011-A-123456-Smith-John-001-v1.pdf
10011-A-123456-Smith-John-002-v1.pdf

When combined in the indicated sequence, the "parts" form a single document.

More examples of file names used for release documents are shown below:

10011-Supplementary-v1.pdf
10011-A-123456-Smith-John-v1.pdf
10011-B-654321-Smith-John-v1.pdf

Where do the PDF files reside?

You cannot choose the save location for the PDF files. The system automatically creates the files in a shared directory path configured by your ROI system administrator. The path includes your OneContent user name as the name of the directory where your output files will reside.

Path format is:  \\<server-name>\<shared-dir$>\<your-username-dir>\

Normally, ROI users do not need to access PDF files once they are created. The files exist only for downstream use by staff personnel who are responsible for shipping the files on physical media.

What does the PDF output include?

For PDF output, each release may consist of multiple PDF files. The release includes one PDF file for each unique request ID-facility-MRN combination that exists on the request and for which documents are being released. If you generate letters and/or invoices for the release, the release also includes a supplementary PDF file that contains the letters and invoice you generated. Requestors can associate supplementary PDF files with the patient PDF files by virtue of the request ID in the file names.

Attachments are a special case

You can print and fax PDF attachments and can save XML attachments as PDF files. Each attachment is generated as a single PDF file that contains the source encrypted XML file and the human-readable PDF file as embedded files. Each container has a unique file name formatted as:

<Request ID>-<Attachment Type>-<Facility ID>-<MRN>-<Patient Name>-<Unique attachment number>.<extension>

For example: 12345-CCR-CCDA-654321-SMITH MARTHA L.-1.pdf
