Potential Patient Mismatch dialog box

The Potential Patient Mismatch dialog box displays when:

  • You create a new request with the patient as the requestor and some of the selected patients do not match the requestor.
  • You have added patients to a request in which the requestor is a patient and the other patient records do not match the requestor.

The dialog box lists patients you selected on the calling form, highlights any potential mismatch(es), and allows you to select no patient, one patient, or multiple patient records to continue.

To open the form

  1. Select Patients > Find Patient and perform a search.
  2. From the search results grid, select more than one patient.
  3. Click Patient Requesting.

Note: This dialog box is also launched from the Requests function at the Add Patient dialog box when the selected patient(s) do not match the requestor.

Item descriptions

Patients grid

Displays an entry for each patient name you selected on the calling form and allows you to select one or more patient records to include on the request.

Initially sorted by Patient Name in alphanumeric ascending order. Can be sorted by any column. By default, the first row is auto-selected.

Information includes:

  • Patient Name
    The name of the patient selected in the calling form. Sort is on the first characters displayed, regardless of which name (first or last) occurs first.
  • DOB
    The date of birth of the patient.
    Initial sort is date ascending. Format is mm/dd/yyyy.
  • SSN
    The patient's Social Security Number.
    Initial sort is numeric ascending.
  • Facility
    The facility where patient care occurred.
    Initial sort is alphanumeric ascending.
  • MRN
    The patient's medical record number.
    Initial sort is alphanumeric ascending.
  • EPN (displays only for EPN systems)
    The patient's enterprise person number.
    Initial sort is alphanumeric ascending.
    Displays only if the dialog box was launched by the Add Patient function when maintaining requests.

Continue with Selected Patient(s)

Creates a new request that includes records for all the patients you select in the Patients grid and associates the request with the currently selected patient requestor.

Enabled only when at least one patient is selected.
