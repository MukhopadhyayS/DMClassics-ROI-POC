About attachments

Patients often request an electronic copy of their health information (for example, information about procedures, diagnostic test results, problem lists, medication lists, list of allergies, and so on). These records, collectively referred to as Clinical Summaries (Continuity of Care Records (CCRs) and Continuity of Care Documents (CCDs)), can be associated with a patient's OneContent record as "attachments."

The records can be acquired as encrypted files from a local file system or acquired electronically from originating clinical systems. During the attachment process, the system copies the original files to a storage folder in the OneContent file system and secures them against access by unauthorized users.

You can add attachments to ROI requests like other patient documents. The release of attachments and their corresponding number of pages are tracked in the Accounting of Disclosures report and are included in the Release History for the request.

You can add attachments to the patient record from these forms:

  • Patients > Patient Records
  • Requests > Patient Information
Attachment file types

Currently, only Clinical Summaries can be attached to the patient record. They exist as XML and/or PDF files. The XML file is an encrypted machine-readable file, and when you attach it to a patient record, the ROI Module outputs the file content to PDF format.

Attachment metadata

When you add attachments to a patient record, the ROI Module captures file information (some by automatic means and some by entries you make). Once the files are released, the attachment and its metadata cannot be changed or deleted.

File metadata fields are:

  • Facility and MRN
  • Attachment Type and Date
  • File name, including the extension
  • Number of pages (XML files are always one page; PDF files reflect the actual number of pages in the document)
  • Title
  • Document source (that is, the source system or assigning authority)
  • Encounter number, service date, and comments

Releasing attachments

Like other patient documents, attachments can be added to a request. You can also print, fax and email PDF attachments, and you can save XML attachments as PDF files.  

Auditing

Auditing is controlled by OneContent audit settings for each facility.  

If the corresponding codes are enabled in OneContent, the system audits these events:

  • Adding attachment files to the system (code N must be enabled)
  • Indexing a document (code Q must be enabled)
  • Viewing attachments (code V must be enabled)
  • Deleting attachments (code X must be enabled)

The system also audits the output method that is used on the request Shipping tab:

  • Print output method (code RP must be enabled)
  • Save as File (export to PDF) output method (code RD must be enabled)
  • Fax output method (code RF must be enabled)

The system captures the event code, the user ID, the facility ID, the patient's MRN and/or encounter number, the date and time of the event, and a remark identifying the document type and filename.
