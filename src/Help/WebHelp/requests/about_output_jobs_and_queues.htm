About output jobs and queues

Output jobs are created when you elect to print, fax, email or export to PDF format a letter, invoice, or other document.

The output service module enables you to monitor and manage output jobs and queues.

For jobs, authorized users can:

  • View all active output jobs (that is, jobs that have not been purged)
  • View a list of all jobs created (no matter which queue they were sent to)
  • Resubmit jobs
  • Filter, sort, or refresh the job list
  • Pause, resume, and cancel jobs

For queues, authorized users can:

  • View a list of all queues
  • View a list of jobs for a specific queue
  • Hold or release queues
  • Filter, sort, or refresh the queue list
  • Select a specific queue to list the jobs it contains

The system also displays status information from integrated third-party print, fax, and PDF products. For example, if an outbound fax returns a specific error (such as for a busy line or when a person answers the phone), the system displays the error in the Output Job Monitoring window.

In addition, the system automatically purges completed jobs and retries failed jobs until the retry period expires. If the job still fails, the system sets its status to Error.

Permissions
  • All ROI permissions are set in the OneContent application.
  • An ROI administrator user can monitor and act on all output jobs for the entire enterprise.  
  • A non-administrator ROI user can monitor or act on only those jobs submitted by that user.   
Job lists

To maintain an up-to-date list of jobs, you can manually refresh the list on-demand.

The system refreshes the list automatically at an interval that is configured on the server (default is 60 seconds), and after an output event occurs, such as when a job fails to complete or after a job is resubmitted, canceled, paused, resumed, completed, or added.

Job information

All users can view information for each job in a queue.

Job information includes:

  • Status. The current status of the job, such as Ready, Paused, Failed, or Processing. The job status is automatically updated based on output events.
  • Type. The job type (Print, Fax, or File).
  • Date Submitted. The date and time the job was submitted.
  • Job Name. The name of the job (<app-name>.<request-ID>).
  • Pages. The number of pages that have printed/faxed and the total number of pages in the job, including letters, separator sheets, and so on.
  • Queue. The name of the queue where the job resides.
  • Job ID. Job ID generated by the system for each job.
  • User ID. The user name or ID of the person who generated the job.
Queue information

Only administrator users can view queue information. Queues are filtered based on your permissions and on the selected filter criteria (Print, File, Fax, or Email).  

The system displays the following information about queues.

  • Type. The type of queue (Print, Fax, Email or Save as File).
  • Status. The current status of the queue (such as Active or Held).
  • Queue. The name of the queue as defined when the queue was created.
  • Description. The description given to the queue.
  • Jobs. The number of jobs on the queue. The number is automatically updated based on output events and is based on your ROI permissions.  For administrators, the total is the actual number of jobs. For a non-admin user, the total represents only the number of active jobs submitted by that user.
