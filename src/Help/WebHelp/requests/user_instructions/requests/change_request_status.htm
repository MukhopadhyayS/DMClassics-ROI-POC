Change request status

This function enables an authorized user to change the request status for a request and/or status reason(s). Some status options are restricted and cannot be manually assigned, but can be changed.

You cannot manually assign the following statuses:

  • Auth-Received. You cannot manually assign this status. The system assigns this status when you create a skeleton request by scanning, indexing, and releasing an authorization request form. A workflow thread automatically assigns the status.

If you have permissions, you can change this status to Logged, Pended, Denied, or Canceled, or to Completed (if at least some documents have been released).

  • Auth-Received Denied. You cannot manually assign this status. The system assigns this status when you create a skeleton request by scanning, indexing, and releasing an invalid Auth-Received request form, which has a document type that is different from valid authorizations. A workflow thread automatically assigns the status.

If you have permissions, you can change this status to Logged, Pended, or Canceled, or to Completed (if at least some documents have been released).

  • Pre-Billed. You cannot manually set this status. The system assigns this status when you click the Pre-Bill button on the Billing & Payment form.

If you have permissions, you can change this status to Logged, Pended, Denied, or Canceled (if you have permissions), or to Completed (if at least some documents have been released).

A special situation exists when a request displays a status of "Unknown." In those cases, the system cannot automatically determine the status of the request. You must ascertain the status and manually assign it to the request.

Note: You cannot set the status to Denied or Canceled if even one document has been released for the request. If you set the status to Canceled or Denied, no further changes (such as adding or removing documents) can be made to the request. However, users will be able to add comments, generate letters, or change the status again. If you cancel or pend the request, the system automatically writes off any balance due.

1. Find a request.
2. On the Request Information form, click Update Request Status. The system displays the Update Request Status dialog box.    
3. Complete the Update Request Status dialog box.
4. At Status, select a status option.  Valid request statuses are:
- Logged. The system assigns this status to a new request or when you retrieve a request that has an Auth-Received status. It indicates that the request requires attention. You can manually assign this status to a request and, if you have permissions, can change it to Pended, Denied, or Canceled, or to Completed (if at least some documents have been released).
- Pended. This option is available only if you have permission to pend a request. When you assign this status, the system prompts you to create a letter to the requestor. You can change this status to Logged, or to Denied or Canceled (if you have permissions), or to Completed (if at least some documents have been released).
- Denied. This option is available only if you have permission to deny a request and only if no documents have been released. When you assign this status, the system sets balances to zero and prompts you to create a letter to the requestor. You can change this status to Logged, or to Pended or Canceled (if you have permissions), or to Completed (if at least some documents have been released). Requests with a Denied status display information in read-only mode, and permit you to only generate letters, maintain comments, or change the status.
- Canceled. This option is available only if you have permission to cancel a request and only if no documents have been released. When you assign this status, the system sets balances to zero and prompts you to create a letter to the requestor. You can change this status to Logged, or to Pended or Denied (if you have permissions), or to Completed (if at least some documents have been released). Requests with a Canceled status display information in read-only mode, and permit you to only generate letters, maintain comments, or change the status.
- Completed. Indicates that at least some of the requested documents have been released (the request cannot assume Completed status if no documents have been released). You are not required to set the status to Completed after all documents have been released. In addition, the system alerts you when one or more patients on the request have no documents specified for release. You can manually change this status to Logged or Pended.

The system displays status options based on your permissions and on the current request status.

5. Provide a reason for changing the status. You can select one or more standard reasons, or, to create a free-form reason, click Add Freeform Reason and enter text.
6. Click Save.
7. On the Request Information form, click Save to complete the event.
