Release documents

This task enables an authorized user to build a new request, release documents to the requestor, and generate a cover letter and invoice.

The system audits the event only when the "R" (ROI Released) audit code is enabled in OneContent.

1. Select Requests > Find / Create Request.
2. Click New Request. The system displays the Request Information form.   
3. Enter information on the Request Information form.
4. Select a request reason and specify the request date.
5. Click Select Requestor, enter data in the Select Requestor dialog box, and return to the Request Information form.
6. Click Save & Continue.
7. Enter information on the Patient Information form.
8. Click Add Patient, add one or more patients to the request, and return to the Patient Information form.
9. Select a patient you just added. At the document tree view, select the document(s) you want to release for this patient and click Add to add them to the Document(s) Selected for Release box.
10. Repeat for the next patient, if one exists.
11. Click Save & Bill to continue building the request.
12. On the Release & Invoice form, enter the billing and shipping information, select an output method, and apply/unapply sales tax.
13. If no pre-bill is required, click Release Invoice. (Otherwise, see Pre-bill a request for further instructions.)

Note: The Release & Invoice and Pre-Bill buttons are disabled if unsaved changes exist. Either save or revert the changes to enable the buttons. The Re-Release button is enabled only if you selected previously released documents in the DSR document tree on the Patient Information form.

14. At the Release dialog box, specify a cover letter template and an invoice template, and click Continue.
Tip: If the requestor is exempt from invoicing, select None at the Invoice Template field to avoid generating an invoice for this request. The option displays for only eligible requestors.

When the system displays the Preview of Release Letter(s) dialog box, click Continue. After output is successfully generated, the system returns you to the Release & Invoice form.

15. The release process is complete within the ROI system. Complete any manual steps (such as handing off to the shipper or communicating the Request Password and/or Queue Password to the requestor). You will be able to access this request later to record payments, to re-send or re-bill, or enter notes as needed.
