About printing, faxing, saving, and emailing documents or writing to disc

The ROI Module displays patient documents stored in the OneContent database through the WinDSS document viewer that is provided with the base OneContent product, and you can use the WinDSS view and print functions as needed. See online help available through the WinDSS application for user instructions.

The ROI Module uses its own output service to print, fax, or save as a PDF file all letters, invoices, pre-bills, statements and patient documents (including those rendered in the WinDSS document viewer) that are selected or generated for release. The output service can also write information to disc (such a CD or DVD).

Note: When viewing an authorization document, you can only print it (not fax or save it) and cannot print any other document.

If you elect to use an output method that is invalid for any of the items being released, the system informs you and allows you to adjust your actions. You may also modify properties such as fax numbers or the number of copies to print that are governed by the output option you choose.  

About print, fax, or email output

These output methods are effective with any printable document including documents, supplemental documents, invoices, pre-bills, letters, statements, and attachments.

Documents are output in the following order:

• Cover letter (if selected)
• Invoice letter (if selected)
• Documents selected for release.

The event is audited if the following facility audit codes are enabled in OneContent:

• RP - Printed
• RF - Faxed
• EM - Email
About output to a PDF file

This is effective with any machine-readable or non-PDF attachments or release documents.

The event is audited if the following facility audit code is enabled in OneContent:

• RD - Export to PDF
• Disc

This applies to third-party devices such as Rimage that allows for writing of information to CD or DVD.

The event is audited if the following facility audit code is enabled in OneContent:

• RC - CD/DVD

See About PDF files for more information.

Patient identifiers in headers and footers

The output service supports printing headers, footers, and watermarks on each page of a released document, using metadata specified by your system administrator. These features are enabled in the ROI Administration function by an authorized user and apply to all released documents. When headers and footers are enabled, the configured patient identifiers are printed on each document page that is released. They are not included on invoices, cover letters, certification letters, other letters, or page separators.

The following patient identifiers are applied to release document pages: Patient Name, Patient MRN, Patient Encounter, and EPN (for sites that are EPN-enabled).

Output queues

The system remembers across application sessions the last queue you used for each type of output and uses it as the default output queue for the next output request of the same type.

Permissions

Your OneContent security permissions control which output options you can use. However, the ROI Module does not enforce your OneContent facility-level print/fax security right when you are viewing documents, releasing documents, or generating letters, invoices, pre-bills, statements and reports.

If you cannot perform an operation, you most likely do not have appropriate permissions, and you should consult your system administrator.

  • To print or fax, you must have the ROI Print/Fax permission.
  • To email, you must have the ROI Email permission.
  • To save output to a PDF file, you must have the ROI Export to PDF permission.
  • To write to disc, you must have the ROI CD/DVD permission.
