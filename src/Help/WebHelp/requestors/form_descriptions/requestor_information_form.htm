Requestor Information form

The Requestor Information form enables authorized users to:

  • Create non-patient requestors. In this case, all fields are blank or use a default value.
  • Create patient requestors. Depending on how you arrive at the form, fields may be pre-populated with existing patient information.
  • View and maintain information for an existing requestor.
  • Create requests.

To open the form

  1. Select Requestors > Find Requestor.
  2. Do either of the following:
- Click New Requestor to create a requestor.

OR

- Search for and select a requestor. Then click View/Edit Requestor to view or edit the requestor's current information.

Item descriptions

Pre-Payment Required

When checked, the requestor must pre-pay the request before it is fulfilled.

Certification Letter Required

When checked, a certification letter is required for this requestor.

Type

Displays a list of requestor types configured for your system.

For a new requestor, defaults to Select Requestor Type (or to a patient requestor type, depending on how you arrive at the form) and can be changed until you save the record. Cannot be changed for an existing requestor.

Inactive

When checked, indicates that the requestor is currently inactive.

Name

Allows you to enter, view, or edit the name of the requestor. For a non-patient requestor type, the system displays only one box labeled "Name." For a patient requestor type, the system displays two separate boxes: Last Name and First Name.

Unique names are not required. The system alerts you to duplicate names and prompts for confirmation to save anyway. The name cannot exceed 256 characters.

Patient EPN

Enterprise Person Number of the patient. EPN is an identifier created by an Enterprise Master Person Index (EMPI) system to uniquely identify a person across all enterprise systems and facilities.

Displays only when the Requestor Type is Patient AND the system is EPN-enabled. Optionally includes the EPN prefix, depending on your election (default setting is no prefix).

Patient DOB

The patient requestor's date of birth.

Enabled only when Requestor Type is Patient.

For a new requestor, defaults to Select Date.

Patient SSN

The patient requestor's Social Security Number.

Enabled only when the Requestor Type is Patient.

Patient Facility

The patient requestor's Facility where patient care occurred.

Enabled only when Requestor Type is Patient.

For a new requestor, defaults to Select Facility.

Patient MRN

The patient requestor's MRN (Medical Record Number).

Enabled only when Requestor Type is Patient.

Main Address group box

The requestor's main address. Contains up to three address lines, city, state/province/region, zip/postal code, and country information. (Click the drop-down list to override the default country.)

Always enabled and editable.

Alternate Address group box

The requestor's alternate address. Contains up to three address lines, city, state/province/region, zip/postal code, and country information. (Click the drop-down list to override the default country.)

Always enabled and editable.

Contact Information group box

Requestor's contact information.

Contains fields for the requestor's home, work, and cell phone numbers, requestor email address, fax number, and the name, phone number, and email address of the contact person for the requestor.

Always enabled and editable.

Save

Saves changes to the requestor's information.

Enabled only when the Type and Name fields contain a value and unsaved data exists.

Performs field validation and displays alerts or messages if problems exist.

If requestor name is a duplicate, prompts whether to save anyway and continue.

Revert / Cancel

Discards unsaved data.

If you are viewing existing requestor information:

  • Button label is Revert
  • Enabled only if you change data
  • Discards unsaved data and returns field values to the last saved state

If you are creating a new requestor:

  • Button label is Cancel
  • Is always enabled
  • Discards unsaved data

Delete Requestor

Deletes the currently selected requestor from the database. Prompts for confirmation.

For a new requestor, enabled only after you save the requestor.

For an existing requestor, displays and is enabled only if the requestor is not currently included on a request.

Create Request

Allows you to create a new request for the selected requestor.

Displays only if you have Create Request permissions.

If you are creating a new requestor, enabled only after you save the requestor.

For an existing requestor, if unsaved changes exist, prompts for confirmation to continue.

If the requestor is a non-patient requestor, launches the Request Information form and inserts the current requestor as the requestor on the new request.

If the requestor is a patient requestor, launches the Create New Request with Patient Requestor dialog box.
