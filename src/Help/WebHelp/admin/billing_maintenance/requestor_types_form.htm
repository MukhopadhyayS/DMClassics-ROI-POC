Requestor Types form

The Requestor Types form allows you to view, add, modify, or delete requestor types such as Patient, Attorney, or Internal Auditor.

Requestor types are used when creating or changing ROI requests. They are associated with system (electronic) billing tiers and can also be associated with billing templates.

To open the form

Select the Administration tab > Billing Maintenance > Requestor Types.

Item descriptions

Main Content Pane

Requestor Types grid

Displays the requestor types and associated information currently defined in the system. The grid title is preceded by the number of existing requestor types.

The Requestor Types grid contains the following columns:

  • Icons that indicate the type, or source, of the requestor type data:
• The system icon indicates that the requestor type is system defined. You cannot delete system-defined requestor types or change their names.
• The user icon indicates data that is sample data or is user created. You can delete the requestor type or change the associated information.
• Requestor Type

Names of the requestor types currently defined in the system.

• Record View

Record views associated with the requestor types.

• MPF Billing Tier

Electronic billing tiers associated with the requestor types.

• Non-MPF Billing Tier

Billing tiers associated with the requestor types.

• Sales Tax

Indicates if sales tax is applied to this item by default.

• Billing Template

Billing templates associated with the requestor types (optional).

Create New

Allows you to enter the information needed to add a new requestor type.

To create a new requestor type, at least one system (electronic) and one non-system billing tier must already be defined in the system. They can be the same billing tier or different billing tiers.

Delete

Deletes the selected requestor type.

Note: You cannot delete system-defined requestor types. They are indicated by the system icon in the Requestor Types grid. You also cannot delete a requestor type that has requestors associated with it.

Detail Pane

Requestor Type

Specifies the name for a requestor type. Requestor type names must be unique. They are not case sensitive.

You can change the names of user-defined requestor types at any time. These requestor types are indicated by the user icon in the Requestor Types grid.

You cannot change the names of system-defined requestor types. These requestor types are indicated by the system icon in the Requestor Types grid.

Default Record View

Specifies the default record view for the requestor type. The system lists the record views that belong to the record view group assigned to the User Profile in the OneContent system. For information, see the OneContent online help.

MPF Billing Tier

Specifies the system (electronic) billing tier for the requestor type.

To define a requestor type, at least one billing tier with the electronic media type must already be defined in the system.

Non-MPF Billing Tier

Specifies the non-system billing tier for the requestor type.

Accommodates the ability to charge differently for non-system documents. (Non-system documents are documents that have no document image in the system database.)

Sales Tax

Indicates if sales tax is applied to this item by default. Check to select this option. The default is unchecked.

Invoice Optional

When enabled, allows the user to release documents without sending an invoice. Unchecked by default.

Billing Template

Allows you to associate fee types with a requestor type.

Assigning a billing template is optional. You can define information for the following items:

• Assign: Specifies one or more billing templates to associate with the requestor type.
• Default: Specifies the default billing template when more than one has been assigned.

Save

Saves the new or modified requestor type.

Cancel

Cancels the requestor type you were adding or any unsaved changes you made to an existing requestor type.
