Maintain authorization request documents

Currently, this feature is unavailable.

Only one document type in a code set can be designated as an authorization request document; it triggers the creation of a request within the ROI Module. To designate a document type as an authorization request document for purposes of the ROI Module, do the following:

  1. Select Administration > Other Maintenance > Document Types.
  2. On the Document Types form, select the Code Set you want to maintain.
  3. In the Document Types grid, select one and only one document type in the Auth Request column.
  4. Click Save.

Note: Code sets are maintained in the OneContent application. For information, see the OneContent online help.
