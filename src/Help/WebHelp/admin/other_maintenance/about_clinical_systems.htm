About clinical systems

HIM departments must sometimes supplement the patient chart with a snapshot of the patient's clinical history. This is accomplished primarily by acquiring summary documents from a clinical system. The OneContent system can be integrated with these clinical systems, making it easy for ROI Module users to download the documents and include them on an ROI request.

About integrated systems and clinical system profiles...

Currently, the ROI Module supports integration with two types of clinical systems: Allscripts Paragon systems and Horizon Clinicals systems. Integration is performed by OneContent experts during the installation of the ROI Module. It defines vital system touch points that support the acquisition of documents and turnaround time reporting. Transaction layouts conform to the requirements of the originating clinical system and cannot be changed.  

A clinical system profile is configured within the ROI Module for each OneContent facility. Then, in the OneContent application, each OneContent facility is associated with its clinical system profile. If connection settings for a facility's clinical system change, ROI Module administrators can easily update the settings within the module.

About acquiring clinical documents...

ROI Module users can "pull" Clinical Summary documents on demand from a facility's clinical system when the information is requested. On-demand acquisition ensures that the latest snapshot is provided versus a potentially stale copy of the document. A document that was "pulled" but never released is tracked and included in Turnaround Time metrics along with documents that were released.

Documents acquired from an external clinical system are referred to as "attachments."

About storing clinical documents...

When acquired, attachments are automatically associated with the patient's OneContent record, are stored within the OneContent file system, and are listed in the patient record like any other patient document.

About turnaround time and notifications...

Turnaround time measures how quickly a provider issues requested documents to the requestor. The ROI Module automatically calculates turnaround time for all Meaningful Use documents (Clinical Summaries (CCRs/CCDs), Discharge Instructions, and Discharge Summaries), regardless of whether the documents are ever released.  

Turnaround time is measured in hours and minutes and uses the following calculation:  <release-date> - <request-receipt-date> - <weekend-days>.

Metrics

The ROI Module stores turnaround time metrics in the OneContent database. Metrics are important to entities that need to show compliance with government regulations, but not all clinical systems require metrics for all Meaningful Use documents. The metrics can be either pulled from the OneContent database on demand or sent to clinical systems as events occur.

On-demand metrics

The Business Insight™ product can pull the metrics on demand from the OneContent database and act as the presentation layer for a clinical system. This is the method used by the Horizon Clinicals products to acquire turnaround time information. Metrics are kept for all MU documents (Clinical summaries, Discharge Summaries, and Discharge Instructions).

Automatic reporting and notifications

The ROI Module can also automatically report the metrics directly to a clinical system. This is the method used by the Paragon product. Currently, turnaround metrics are reported to a Paragon system only when all of the following conditions are true:  

  • A Clinical Summary document has been released for the first time on a request, AND
  • The document was acquired specifically for the request (that is, it was not acquired by an earlier request).

The ROI Module also automatically sends notifications to a Paragon system when:

  • A request that contains a Meaningful Use document is canceled, denied, or deleted.
  • A Clinical Summary document is downloaded from a clinical system but is never attached to a request.

Auditing

Auditing is controlled by OneContent settings for each facility's Audit Trail Setup.

If the corresponding codes are enabled in OneContent, the system audits these events:

  • Configuring external clinical systems (code I must be enabled)
  • Notifications (code 2 must be enabled)

The system captures the event code, the user ID, the facility ID, the patient's MRN and/or encounter number, the date and time of the event, and a remark describing the event. For notifications, the system also captures the notification status (successful or failed).
