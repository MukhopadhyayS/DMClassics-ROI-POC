Request Reasons form

The Request Reasons form allows you to view, add, modify, or delete the reasons that users can select when maintaining an ROI request. Examples of request reasons include Attorney Request, Hospital Request, and Patient Request.

To open the form

Select the Administration tab > Reasons Maintenance > Request Reasons.

Item descriptions

Main Content Pane

Request Reasons grid

Displays the request reasons and associated information currently defined in the system. The grid title is preceded by the number of existing request reasons.

The Request Reasons grid contains the following columns:

  • Reason Name
    Names of the request reasons currently defined in the system.
  • Attribute
    Attributes associated with the request reasons. The attribute can be TPO (related to treatment, payment, or healthcare operations) or non-TPO. This attribute facilitates Accounting of Disclosures (AOD) reporting.
  • Display Text
    Display text associated with the request reasons. This text displays on letters if the attribute has been assigned to the letter template, in the audit trail report if configured for this, and in the system-generated history log.

Create New

Allows you to enter the information needed to add a new request reason.

Delete

Deletes the selected request reason. You can delete request reasons at any time.

Detail Pane

Reason Name

Specifies the name for the request reason.

Request reason names must be unique. They are not case sensitive. You can change them at any time.

Attribute

Specifies the attribute to associate with the request reason.

You can select TPO (related to treatment, payment, or healthcare operations) or non-TPO. This attribute facilitates Accounting of Disclosures (AOD) reporting.

You can change the attribute for a request reason at any time.

Display Text

Specifies the text to display for the request reason. This text displays on letters if the attribute has been assigned to the letter template, in the audit trail report if configured for this, and in the system-generated history log.

Display text must be unique. It is not case sensitive. You can change it at any time.

Save

Saves the new or modified request reason.

Cancel

Cancels the request reason you were adding or any unsaved changes you made to an existing request reason.
