Patient Records form

The Patient Records form provides access to documents for a selected patient. The document tree on the Patient Records form lists all encounters, but filters documents based on your record view permissions and on the document types associated with each record view.

To open the form

  1. Select Patients > Find Patient.
  2. Search for and select a patient. Then click View / Edit Patient.
  3. Select Patient Records in the left pane.

Note: You can access this form at any time while viewing and/or editing patient information.

Item descriptions

Patient information box

Displays the patient's name, gender, date of birth, facility where care occurred, Social Security Number (masked, when enabled), EPN, and MRN. Also contains "VIP" if the patient has VIP status.

Filter by Encounter

Defaults to Off.

Displays On when the system is set to filter the patient's records by a subset of encounters.

Set/Update

Allows you to set or change a filter to display only information related to a set of selected encounters.

Rules are:

  • Not available for non-OneContent patients or records
  • Enabled only for OneContent patients
  • Launches the Set Encounter Filter dialog box

Record View

Allows you to filter documents by selecting a record view.

Lists all record views that you have permission to access and defaults to the first record view option in the list.

Enabled only for OneContent non-global documents.

Only Disclosure Docs

Filters records to show only disclosure documents. The disclosure document is a form completed by the patient to identify one or more entities from which to withhold all or part of the patient's records. The facility cannot release the patient's documents to any entity listed on the form.

Displays only if you have permission to view disclosure documents.

  • Enabled only for OneContent patients and documents
  • Defaults to off
  • When checked, causes the document tree to show only documents marked as disclosure documents

Sort By

Sorts documents according to the option you choose.

Enabled only for OneContent patients and documents. Has no affect on OneContent global documents or non-OneContent documents.

Options are:

  • Encounter (default setting)
  • Displays documents by encounter in chart order
  • Sort order is by Encounter (most recent first), then by document type (in Chart order), then by version, most recent version first (if view versions is enabled), then by page number (ascending)
  • Document
  • Displays documents alphabetically by Document Name by encounter
  • Sort order is by document type (alphanumeric ascending), then by encounter (most recent first), then by version, most recent version first (if view versions is enabled), then by page number (ascending)

Show Versions

Shows document versions in the document tree.

Displays only if you have permission to view document versions.

  • Enabled only for OneContent patients and documents
  • Defaults to off, and the records tree control shows only the page numbers for the most recent version
  • When checked, the records tree control shows all versions of all documents in the record view

Expand/Collapse buttons

Allow you to quickly expand or collapse all nodes in the document tree.

Document tree

Allows you to navigate to and select records associated with the current patient. Always lists encounters, but filters documents based on your record view permissions and on the document types associated with each record view.

For OneContent patients, displays Global, OneContent, non-OneContent, and attachment documents. For non-OneContent patients, displays only non-OneContent and attachment documents.

  • The "GLOBAL DOCUMENTS" folder lists all documents for the selected patient that are classified in OneContent as global. Display is in chart order, most recent first.  Information includes document type name and admit date and time. Display is not affected by Filter, Record View, or Sort By selections.
  • If non-OneContent documents exist for this patient, the tree displays a "NON MPF DOCUMENTS" folder. The node lists documents, most recent first, with document type name, facility, department, date of service, and pages. Display is not affected by Filter or Sort By selections.
  • If attachments exist for this patient, the tree displays an "ATTACHMENTS" folder. The node lists documents that have been attached to the patient.
  • Remaining folders represent OneContent documents and are sorted according to the Sort By option. OneContent documents are grouped by Encounter, Document Names, Version, and Page numbers. At the encounter level, the system displays encounter number, patient status, facility and admit date.

Also identifies encounters with one or more deficiencies () and, if you have adequate permissions, identifies VIP patients () and locked encounters ().

Enc. Details

Displays encounter information for the encounters currently selected in the document tree.

Enabled only when one or more OneContent encounters are selected.

Launches the Encounter Details dialog box.

View Selected

Opens currently selected global and/or a OneContent document or attachment in the document viewer. Non-OneContent documents cannot display in the viewer.

Displays only for OneContent patients.

  • Enabled only after you select the "Global Documents" or "Attachments" node, or at least one Global or encounter, document, page, or version
  • Disabled if only Non-OneContent documents are selected

View All

Opens all global and/or OneContent documents in the document viewer. Non-OneContent documents cannot display in the viewer.

Displays only for OneContent patients.

  • Enabled only after you select at least one Global or OneContent encounter or document
  • Disabled if you select the Global Documents or Non-OneContent folder or a non-OneContent document

New Document

Enables you to create a new non-OneContent document for the current patient.

Launches the Create New Non-MPF Document dialog box.

Edit Document

Allows you to edit a selected non-OneContent document for the current patient.

Enabled only after you select at least one non-OneContent document.

Launches the Edit Non-MPF Document dialog box.

Add Attachment

Allows an authorized user to add an attachment to the patient record.

Launches the Create New Attachment dialog box.

Edit Attachment

Allows an authorized user to edit information for an attachment or delete/withhold an attachment from the patient's record.

Launches the Edit Attachment dialog box.

Patient Requesting

Allows you to create a new request for the selected patient, with the patient as the requestor.

Displays only if you have Create Request permissions.

  • For a new patient, enabled only after you save information to create the new patient
  • For existing patients, enabled when:
  • EPN is not configured and you have selected at least one patient OR
  • EPN is configured and you have selected only one patient
  • Launches the Create New Request with Patient Requesting dialog box

Create Request

Allows you to create a new request for the selected patient.

Displays only if you have Create Request permissions.

  • Enabled only if you select only one patient in the grid; multi-selections are not allowed
  • If unsaved changes exist (you have added non-OneContent document(s) and/or made changes to existing non-MBF documents), prompts for confirmation to discard changes
  • Launches the Request Information form and adds the current patient as a patient on the new request
