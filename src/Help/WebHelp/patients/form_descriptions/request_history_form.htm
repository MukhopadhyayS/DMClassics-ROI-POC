Request History form

The Request History form allows you to

  • View a list of requests and select one to view request information.
  • Add, change, or delete request information, if you have the required permissions.

To open the form

  1. Select Patients > Find Patient.
  2. Search for and select a patient. Then click View/Edit Patient.
  3. Select Request History in the left pane.

Note: You can access this form at any time while viewing and/or editing requestor information after a patient has been selected or created.

Item descriptions

Patient information box

Displays the patient's name, gender, date of birth, the facility where care occurred, Social Security Number (masked, when enabled), EPN, and MRN. Also contains "VIP" if the patient has VIP status.

Filter By Encounter flag

Displays On if the system is currently set to filter the patient's records by a subset of encounters. Otherwise, displays Off.

Set/Update button

Allows you to set or change a filter to display only information related to a set of selected encounters.

Not available for non-OneContent patients or records. Enabled only for OneContent patients.

Launches the Set Encounter Filter dialog box.

Requests grid

Displays requests for the patient selected on the Find Patient form and allows you to a select request.

Can be sorted on all columns. The default sort order is ascending Receipt Date and then ascending Facility.

Note: If a request contains records from facilities that you are not allowed to access, the detail pane lists the request in red text and masks data in the following columns: Requestor Name and Requestor Type. If you select the request, the system disables the View/Edit Request button.

Contains the following information:

  • Last Updated
    Used as the default sort in most recent to oldest order; format is mm/dd/yyyy.
    The date when the request was last updated.
  • Facility
    The facility where patient care occurred.
    Used as the default secondary sort in ascending alphabetical order.
  • Updated By
    The user who last updated the request. Initial sort order is alphanumeric ascending.
    Format is: LastName, FirstName MiddleInitial
  • Status
    The status of the request: Auth-Received, Canceled, Completed, Denied, Logged, Pended, Pre-Billed.
  • Requestor Name
    Displays the requestor name for the request. Initial sort is alphanumeric ascending.
  • Requestor Type
    Displays the requestor's type. Requestor Types are maintained in the ROI application and identify the category of requester, such as attorneys, employers, or insurance. Initial sort is alphanumeric ascending.
  • Request Id
    Displays the ID of the request. Initial sort order is alphanumeric ascending.
  • Receipt Date
    Displays the date the request was received; format is mm/dd/yyyy.
    Initial sort order is oldest to most recent date.
  • Balance
    Displays the total charges (in dollars) for the request. Initial sort order is numeric ascending.

View/Edit Request button

Allows you to view and/or edit a selected request.

Enabled after you select a request. (Remains disabled if you select a restricted request.)

Launches the Request Information form and inserts the selected request into the form.

Patient Requesting button

Allows you to create a new request for the selected patient, with the patient as the requestor.

Displays only if you have Create Request permissions.

  • For a new patient, enabled only after you save information to create the new patient
  • For existing patients, enabled when:
  • EPN is not configured and you have selected at least one patient
  • EPN is configured and you have selected only one patient
  • Launches the Create New Request with Patient Requesting dialog box

Create Request button

Allows you to create a new request for the selected patient.

Displays only if you have Create Request permissions.

Launches the Request Information form and adds the current patient as a patient on the new request.
