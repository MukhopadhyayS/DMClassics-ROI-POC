Logging off

Logoff can occur in one of two ways:

  • Manually when you choose to log off or exit the application.
  • Automatically when the system logs you off because no user activity has occurred within a defined period of time.

During manual logoff or exit events, the system checks whether you have unsaved changes. If you do, the system permits you to:

  • Save changes and continue to log off or exit
  • Discard changes and continue to log off or exit
  • Return to the module and continue your work activities

To log off without closing the module

This logoff method enables you to keep the ROI window open even while you are logged off. This is convenient when, for example, you step away from your workstation for a short time and want to prevent unauthorized access.

Select the Security > Logoff menu path, or press Alt+C+O.

To log off and close the module

This logoff method enables you to log off and close the ROI Module window in one operation. You must re-launch the module when you want to log back in.

Do either of the following:

  • Select the Security > Exit menu path, or press Alt+C+X.
  • Click the ROI window's exit icon ().
