Security

Basics

Your site can use either standard user authentication or Directory Services authentication for user logon. Logon procedures vary depending on the state of your user account and the type of authentication currently in effect for your site. The Directory Services Authentication feature is enabled in the base OneContent system.

  • Prior to Release 15.0, all ROI Module installations used standard authentication. Standard authentication requires you to log on using your OneContent user ID and password. The setting for maximum failed logon attempts also applies to ROI failed logon attempts.
  • Release 15.0 and later installations have the option of using Directory Services authentication, which requires you to log on using your Windows user ID and password. The Windows authentication settings (not the OneContent settings) control the number of permitted failed logon attempts that can occur before your login account is locked.

Rights

ROI security rights (permissions) are granted in the OneContent user profile and apply across the enterprise. The ROI Module and the OneContent application follow the same rules for document searches, document access, and document display based on your OneContent permissions. This means the document tree behaves identically in both applications.

When you grant or revoke security rights for users who are logged in, the old settings apply until they log out and log back in.

For additional information about granting security rights, see the OneContent online help.

Administrator rights

The following security rights are required to be an ROI administrator:

  • ROI Access Application
  • ROI Administration
