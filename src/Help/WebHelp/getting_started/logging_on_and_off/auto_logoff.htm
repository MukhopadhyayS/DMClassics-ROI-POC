Auto-logoff

The ROI Module is designed to automatically log you off when no user activity has been detected for a specified period of time.

The system auto-logoff settings control this event. The settings exist at the enterprise and user levels and are set by your system administrator.

The enterprise-level setting is used as a default if your profile does not specify a time-out value. When your profile specifies a time-out value, it always overrides the enterprise-level setting.

The system displays a message to warn you that you will be logged off and that unsaved changes will be lost. The message includes a count-down timer and allows 30 seconds for your response.

• To cancel the event, click OK or press any key to cancel the event. The system closes the message box and resets the inactivity timer. Work in progress is retained and you can continue as needed.
• To allow the event, no action is required. If you allow the auto-logoff event to occur, the system discards unsaved work (if any exists), logs you off the application, and re-displays the ROI Logon dialog box.
