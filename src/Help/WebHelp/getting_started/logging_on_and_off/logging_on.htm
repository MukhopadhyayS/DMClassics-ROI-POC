Logging on

ROI users must have explicit permission to access the ROI Module (access to ROI functions is controlled within the OneContent application).

Basic ROI permissions allow you to:

  • Launch the application
  • Search for and view patient information
  • Search for and view requestor information
  • Search for and view requests
  • Print, fax, email or save patient documents to a file
  • Generate reports

Additional permissions are required to perform add, change, and delete operations, as well as other operations like cancelling or denying requests.

Each logon attempt is audited, and the same logon ID can have more than one ROI session open at a time.

Logon procedures vary depending on whether your site uses the Directory Services Authentication feature:

  • Pre-Release 15.0 systems always use standard system authentication.
  • Release 15.0 and later systems have the option of using Directory Services authentication. Directory Services authentication allows you to use your Windows credentials for logging on to OneContent applications.

Note: If you are unsure of which release you are using, at the ROI Module window select Help > About from the menu. The system displays ROI release information. If you are using Release 15.0 (or later) and are unsure whether Directory Services Authentication is enabled, follow instructions for the logon dialog box that matches your screen or ask your supervisor or OneContent system administrator.

Follow instructions for the authentication method used by your site.

  • Standard OneContent authentication
  • Directory Services authentication

 

Logon dialog box

The Logon dialog box allows you to enter your user ID and password and authenticate to the OneContent ROI system.

To open the form, do the following:

Launch the ROI Module or select the Security > Login menu option.

Directory Services logon dialog box

Item descriptions

User ID box

Allows you to enter your user ID. Required.

Password

Allows you to enter the password for your user ID. Required. Case-sensitive.

Domain

Allows you to select the applicable domain for your user ID.

Logon errors

If...

Then...

Your logon failed because your User ID and/or password are invalid

The system alerts you and allows you to try again.

Click OK and try again.

Your logon failed because you used a default password for logon

The system alerts you and requires you to change your password.

Click OK to close the message box and continue to the Change Password dialog box.

Your logon failed because:

  • You do not have permission to use the ROI Module
  • You have exceeded the maximum number of logon attempts allowed
  • Your account is locked
  • Your account is inactive
  • The server is down

The system alerts you to the reason, and you cannot log on.

Contact your system administrator.
