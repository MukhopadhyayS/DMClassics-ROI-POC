About logging on or off

Security basics

Your site can use either standard user authentication or Directory Services authentication for user logon. Logon procedures vary depending on the state of your user account and the type of authentication currently in effect for your site. The Directory Services Authentication feature is enabled at the McKesson Patient Folder Workstation.

  • Prior to Release 15.0, all ROI Module installations used standard MPF authentication. Standard authentication requires you to log on using your McKesson Patient Folder user ID and password. The MPF Workstation setting for maximum failed logon attempts also applies to ROI failed logon attempts.

  • Release 15.0 and later installations have the option of using Directory Services authentication, which requires you to log on using your Windows user ID and password. The Windows authentication settings (not the MPF Workstation settings) control the number of permitted failed logon attempts that can occur before your login account is locked.

More about Directory Services Authentication...

Additional Directory Services Authentication features that your site may use include:

  • User Self-Mapping.  This feature permits you to associate your Windows credentials with a McKesson Patient Folder user ID. At least one MPF user ID must be associated with the Windows login you enter. This is a one-time event that occurs if the Windows credentials are not currently mapped to any MPF user ID.

  • Multiple MPF mappings.  If multiple McKesson Patient Folder user IDs are mapped to the Windows account you enter, the system permits you to select the MPF ID to use for your current session. Multiple mappings might exist, for example, if you regularly perform different tasks using different MPF user IDs, each with a unique set of permissions.

 

Logging on

ROI users must have explicit permission to access the ROI Module (access to ROI functions is controlled within the MPF Workstation application).

Basic ROI permissions allow you to:

  • Launch the application

  • Search for and view patient information

  • Search for and view requestor information

  • Search for and view requests

  • Print, fax, email or save patient documents to a file

  • Generate reports

Additional permissions are required to perform add, change, and delete operations, as well as other operations like cancelling or denying requests.

Each logon attempt is audited, and the same logon ID can have more than one ROI session open at a time.

Passwords

When the Directory Services Authentication feature is enabled, you cannot change your McKesson Patient Folder password because the system uses your Windows credentials for security. However, administrators can still reset McKesson Patient Folder passwords.

When your site uses standard MPF authentication, you can change your McKesson Patient Folder password at any time. Occasionally, the system may prompt you to change your password.  

You are prompted to change your McKesson Patient Folder password when:

  • You log on for the first time.

  • Your password has been reset by the system administrator.

  • Your account is locked (such as after a number of unsuccessful login attempts).

Your password must comply with current MPF Workstation security requirements, and if it fails the security checks, the system displays an informative message. When planning your password, ask your system administrator for password rules currently in effect.

Logging off

Logoff can occur in one of two ways:

  • Manually when you choose to log off or exit the application.

  • Automatically when the system logs you off because no user activity has occurred within a defined period of time.
