Set the queue properties for the queue and then select OK group

The output options you can set will vary, depending on the Queue Type.

  • Queue Type: Print

    The attributes that are listed for the Print option will vary, depending on the printer. Typically, they will include options such as the following, but you may have more options, fewer options, or the same options may be named differently. See the documentation for your printer for additional information.

Note: You may need to place your cursor on the right edge of the dialog box and pull it to the right in order to see the vertical scroll bar.

    • Printers
      Specifies the default printer to use when the print option is selected. All printers belonging to the selected queue will be listed.

    • Collate
      Specifies whether to collate the printed pages.

    • Color
      Specifies whether to print the output in color (when available) or gray scale.

    • Number of copies
      Specifies the default number of copies to print.

    • Duplex
      Specifies whether to print on both sides of a page (when available).

    • Page orientation
      Specifies whether to print in portrait or landscape orientation (when available).

  • Queue Type: Save As File

The files are saved in PDF format. The PDF file naming convention is:

<request ID><release sequence><part number><v_version number>

Where:

request ID = A unique identifier that is generated for a request.

release sequence = A 3-digit, incremental number (001, 002, and so on) that indicates a duplicate request so, for example, when a user prints the same request multiple times, the names will be unique.

part number = A 3-digit, incremental number applied to the end of the file name when a PDF is too large to fit on a single CD or DVD. When the PDF fits on a single disc, a part number is not included.  

v_version number = A sequential number appended to the end of a file name if a new file name exactly matches an existing file name. This ensures that existing files are not overwritten.

The Save As File options you can set are the following:

    • CD Size Limit (MB)
      Specifies the maximum size PDF file that can be created when the user selects CD as the output media type.

    • PDF Enterprise Location  
      Provides the network path for saving PDFs when users select the Save as File option. This location must be a shared directory that ROI users can access. Enter the location using the following UNC path name format:

\\<server-name>\dir$\<ROI-PDF-dir>\

The system automatically creates folders using the user's unique MPF user name as the folder name.

Important: You must notify users of the network path you specify so they can find the generated files.

    • DVD Size Limit (GB)  
      Specifies the maximum size PDF that can be created when the user selects DVD as the output media type.

    • Default PDF Media
      Specifies the default media type in which the PDF file will be saved. The options are CD, DVD, or Other/Unlimited Size. The selected media type will determine how large the PDF file should be and will create multiple PDF files if needed.

    • Default Password
      Specifies the default password to use when the Password Required for PDF Files option is set to Yes.  

Important: If you specify a default password, you must give it to the users that will need it.

    • Password Required for PDF Files: Yes  No
      Specifies whether a password will be required for PDFs when a user selects the Save As File output option.

If you select Yes and the user doesn't enter a password, the Default Password is used.

If you select Yes and the user enters a different password, the password the user enters is used.

If you select No and the user enters a password, the password the user enters is used.

Important: Users must give the password to the requestor so he or she can open the file when received.

    • AutoPurge of PDF Files <number of> hours
      Specifies the number of hours a PDF should be saved for users to access.

  • Queue Type: Fax

    • Queue
      Specifies a name for the RightFax queue.

    • Password
      Specifies the password for the RightFax queue.
