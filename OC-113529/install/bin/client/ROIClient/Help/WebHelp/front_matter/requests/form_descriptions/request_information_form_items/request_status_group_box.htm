Request Status group box

Contains the following items.

  • Current Status

For existing requests, displays the current status of the request.

For new requests, is blank until after the request is saved, and then defaults to Auth-Received (or Auth-Received Denied) if the system created the request or to Logged if a user created the request.

Can be changed by an authorized user.

  • Current Reason

The reason(s) associated with a status change (if the status requires a reason to be given).

Displays the latest recorded reason(s) from the Change of Status and the Status Reason Added events in the event history.

Can be changed by an authorized user.

  • Update Request Status button

Allows an authorized user to change the request status and/or reason(s) for this request.

    • The system automatically assigns the Auth-Received, Auth-Received Denied, and Pre-Billed status. You cannot manually assign these statuses.

    • To update the status of a request, you must have permissions to create and/or update an ROI request, but these permissions authorize changing the status to only Completed or Logged.

To change status to Pended, Denied, or Canceled,  you must have explicit permission to pend, deny, or cancel requests. The system displays only the options you have permission to use.

Launches the Update Request Status dialog box.
