About logging on or off

Security basics

The McKesson Patient Folder Workstation application controls login security for the ROI Module.

  • ROI users must have an MPF Workstation user ID and password.

  • The MPF Workstation setting for maximum failed logon attempts also applies to ROI failed logon attempts.

  • ROI users must have explicit permission to access the ROI Module (access to ROI functions is controlled within the MPF Workstation application).

  • The system automatically logs you off the ROI Module if the defined auto-logoff period elapses with no user activity; the auto-logoff setting exists at both the enterprise and user levels, and the user profile setting always takes precedence.

Basic ROI permissions allow you to:

  • Launch the application

  • Search for and view patient information

  • Search for and view requestor information

  • Search for and view requests

  • Print, fax, or save to a file patient documents

  • Generate reports

Additional permissions are required to perform add, change, and delete operations, as well as other operations like cancelling or denying requests.

Passwords

Your password must comply with current MPF Workstation security requirements. If your password fails the security checks, the system displays an informative message.

You can change your password at any time. You are required to change your password after logging on with a temporary password (such as when you are a new user or the administrator has reset your password). When planning your password, ask your system administrator for password rules currently in effect.

Logging on

Logon procedures vary depending on the state of your user account. Standard logon procedures apply when your user ID and password are valid. Other logon procedures apply depending on whether:

  • This is your first logon as a new user in the system

  • The system administrator has reset your password because your account is locked or your password has expired

Each logon attempt is audited, and the same logon ID can have more than one ROI session open at a time.

Logging off

Logoff can occur in one of two ways:

  • Manually when you choose to log off or exit the application

  • Automatically when the system logs you off because no user activity has occurred within a defined period of time

Thirty seconds before auto-logoff occurs, the system displays a notification message with a countdown timer and allows you to interrupt the logoff action. If auto-logoff is not interrupted, the system discards all unsaved work, closes all open dialog boxes and application windows, and re-displays the ROI Logon dialog box.

If you manually log off, the system prompts whether to save work in progress, if any unsaved entries exist.
