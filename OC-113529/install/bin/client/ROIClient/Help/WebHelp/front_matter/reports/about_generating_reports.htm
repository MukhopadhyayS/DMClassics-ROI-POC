About generating reports

The ROI Reports function provides the ability to generate reports related to ROI requests. This functionality is available by selecting the Reports tab on the left pane of the ROI interface. To generate reports, the Crystal Reports product must be installed and running.

Be aware that printing longer reports can tie up your system for some time. This is largely dependent on your system resources and the date range you specify (more data takes longer).

For information about request statuses, see About requests.

Report options and functionality

For most reports, you can select report criteria, specify a date range, and select the level of detail. You can also print the reports and export (save) them to a file, provided you have the appropriate security rights. The available file formats for saving reports are PDF, XLS, RTF, CSV, and HTML.

You cannot fax reports directly from the ROI interface, but you can save the generated reports to a file and attach the file to an email.

Security rights

You must have the ROI Reports security right to generate ROI reports. See your administrator if you need this right. If you are logged into ROI when the right is granted, you must log out and log back in for the change to take effect.

In addition, if you do not have VIP permissions, the system displays VIP patient names in reports but omits all other information related to those patients. Reports that can display VIP names include Accounting of Disclosure, Documents Released by MRN, Request Detail, and Requests with Logged Status reports.
