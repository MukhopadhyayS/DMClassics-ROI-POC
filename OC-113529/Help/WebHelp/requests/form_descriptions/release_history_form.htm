Release History form

The Release History form enables you to view a list of all document pages that have been released for the currently selected request. You can view a list of all patients or filter the view by patient name, user name, and/or release date.

To open the form

  1. Select the Requests tab.
  2. Search for and select a request.
  3. Click View/Edit Request.
  4. Select the Release History function.

Note: You can also access this form after you save a new request.

Item descriptions

Note: Date and time format are controlled by the regional settings of your operating system.

Requestor information box

Identifies the requestor type, the requestor's name and demographic information, and the requirements for pre-payment and certification letters (if any). Also displays the request ID, current status, and receipt date and time.

All Patients

Displays all release history records.

Selected by default.

Filter Patients

Enables the Patient list box and allows you to select one or more patients for whom to display release history.

Patients are listed in alphanumeric ascending order. If multiple patients on the list have the same name, the system uses the patient identifier (EPN or Facility (1st)/MRN (2nd)) to sort patients in alphanumeric ascending order.

If you select only one patient from the list box, information for that patient displays to the right of the list. If you select more than one patient, patient information is blank.

Patient information fields

Displays information for the patient currently selected in the adjacent list box. Read-only. Information includes:

  • Name
    For a new request with no patients added, displays None Selected.

For an existing request or if at least one patient has been added, displays the selected patient’s name.

Displays the icon when the patient is a VIP or has a VIP encounter.

Displays the icon when the patient is Locked or has a Locked Encounter.

  • Gender
    Blank for a new request with no patients, or if it is missing from an existing request. Otherwise, displays the patient's gender.
  • DOB
    Blank for a new request with no patients, or if it is missing from an existing request. Otherwise, displays the patient's date of birth.
  • SSN
    Blank for a new request with no patients, or if it is missing from an existing request. Otherwise, displays the patient's SSN (may be masked).
  • Facility
    Blank for a new request with no patients, or if it is missing from an existing request. Otherwise, displays the facility where patient care occurred.
  • MRN
    Blank for a new request with no patients, or if it is missing from an existing request. Otherwise, displays the patient's MRN.
  • EPN
    Displays only for EPN systems. Blank for a new request with no patients, or if it is missing from an existing request. Otherwise, displays the patient's EPN.

User Name

Allows you to filter release history by selecting a specific user.

The list box includes only users who have released documents.

Defaults to All.

Release Date

Date/time documents were released. Allows you to filter release history by specifying a date or date range.

Options are:

  • All to return all entries that match the other search criteria, regardless of date
  • 30 days, 60 days, or 90 days previous to the current day; returns only entries that fall within the period you select and that match the other search criteria
  • Custom Range to enable From and To fields where you can enter a date range of your choosing

Defaults to All.

Release History grid

Lists all release events that match the filter criteria. Also displays the total number of release events and the total number of pages released.

Read-only.

Default sort is Date | Time in most recent to oldest order. Can be sorted on all columns. Includes the following information.

  • Date | Time
    The date and time when the document was released.
  • Patient Name
    Patient name associated with released document.
    Used as secondary sort with default. Initial sort order is alphanumeric ascending.
  • Encounter
    Encounter associated with released document.
    Initial sort order is alphanumeric ascending.
    Self-pay indicates the primary payor is the patient.
  • Document - Version - Subtitle
    Name of the document released, including the version of the document and its subtitle.
    Initial sort order is alphanumeric ascending.
  • Pages
    Description of the actual pages released. Example: Pages 1-4, 6, 10-12.
    Initial sort order is numeric descending.
  • Request Password
    Generated by the system and is specific to the request.
    Applies to all documents and attachments that are output as PDF files for this request and displays only when queue passwords are applied to the output.
    Cannot be changed and must be manually communicated to the requestor.
  • Queue Password
    Displays the password specified for the queue by default or by the user. A default value is specified by the ROI Administrator, but can be overridden on the Save As File dialog box.
    Applies to all documents in the queue. Must exist to encrypt the PDF file.
    When specified, always displays.
  • Shipping Address
    Shipping address specified at the time of the release.
    Initial sort order is alphanumeric ascending.
  • Shipping Method
    Method used to ship the released documents to the requestor.
    Initial sort order is alphanumeric ascending.
  • Shipping Tracking Number
    Tracking number issued by the shipping vendor associated with the release.
    Initial sort order is alphanumeric ascending.
  • User Name
    The full name of the user who released the document.
    Initial sort order is alphanumeric ascending.
