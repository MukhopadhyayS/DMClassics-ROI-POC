Cancel, pend, or deny a request

This function enables an authorized user to set the status for a request to Canceled, Pended, or Denied.

The system displays status options based on your permissions and on the current status of the request, and you cannot set the status to Denied or Canceled if even one document has been released for the request.

Note: If you set the status to Canceled or Denied, no further changes (such as adding or removing documents) can be made to the request. However, users will be able to add comments, generate letters, or change the status again.

  1. Find a request.

  2. On the Request Information form, click Update Request Status.    

  1. On the Update Request Status dialog box:   

    1. At Status, select Canceled, Pended, or Denied, as desired. If you cancel or pend the request, the system automatically writes off the balance due.

    2. Provide a reason for changing the status. You can select one or more standard reasons, or, to create a free-form reason, click Add Freeform Reason and enter text.

    3. Click Save.

  1. On the Request Information form, click Save to complete the event.
