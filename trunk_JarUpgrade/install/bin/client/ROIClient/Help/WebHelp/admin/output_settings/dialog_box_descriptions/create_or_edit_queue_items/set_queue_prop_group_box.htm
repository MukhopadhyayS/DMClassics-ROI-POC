Set the queue properties for the queue

The output options you can set will vary, depending on the Queue Type.

  • Queue Type: Print

    The attributes that are listed for the Print option will vary, depending on the printer. Typically, they will include options such as the following, but more options, fewer options, or the same options with different names may exist. See the documentation for your printer for additional information.

Note: You may need to place the cursor on the right edge of the dialog box and pull it to the right to display the vertical scroll bar.

    • Printers
      Specifies the default printer to use when the print option is selected. All printers belonging to the selected queue will be listed.

    • Collate
      Specifies whether to collate the printed pages.

    • Color
      Specifies whether to print the output in color (when available) or gray scale.

    • Number of copies
      Specifies the default number of copies to print.

    • Duplex
      Specifies whether to print on both sides of a page (when available).

    • Page orientation
      Specifies whether to print in portrait or landscape orientation (when available).

 

  • Queue Type: Save As File

The files are saved in PDF format. The PDF file naming convention is:

<request ID><release sequence><part number><v_version number>

Where:

request ID = A unique identifier that is generated for a request.

release sequence = A 3-digit, incremental number (001, 002, and so on) that indicates a duplicate request so that, for example, when a user prints the same request multiple times, the names will be unique.

part number = A 3-digit, incremental number applied to the end of the file name when a PDF is too large to fit on a single CD or DVD. When the PDF fits on a single disc, a part number is not included.  

v_version number = A sequential number appended to the end of a file name if a new file name exactly matches an existing file name. This ensures that existing files are not overwritten.

The Save As File options you can set are the following:

    • PDF Enterprise Location  
      Provides the network path for saving PDFs when users select the Save as File option. This location must be a shared directory that ROI users can access. Enter the location using the following UNC path name format:

\\<server-name>\dir$\<ROI-PDF-dir>\

The system automatically creates folders using the user's unique MPF user name as the folder name.

Important: You must notify users of the network path you specify so they can find the generated files.

    • AutoPurge of PDF Files <number of> hours
      Specifies the number of hours a PDF should be saved for users to access.

    • Password Required for PDF Files: Yes  No
      Specifies whether a password will be required for PDFs when a user selects the Save As File output option.

If you select Yes and the user doesn't enter a password, the Default Password is used.

If you select Yes and the user enters a different password, the password the user enters is used.

If you select No and the user enters a password, the password the user enters is used.

Important: Requestors must know the password to be able to open the files, and users should follow guidelines developed by your organization for conveying either the request or queue password to a requestor.

    • Default Password
      Specifies the default password to use when the Password Required for PDF Files option is set to Yes.  

Important: If you specify a default password, you must give it to the users who will need it.

    • Default PDF Media
      Specifies the default media type in which the PDF file will be saved. The options are CD, DVD, or Other/Unlimited Size. The selected media type will determine how large the PDF file should be and will create multiple PDF files if needed.

    • CD Size Limit (MB)
      Specifies the maximum size PDF file that can be created when the user selects CD as the output media type.

    • DVD Size Limit (GB)  
      Specifies the maximum size PDF that can be created when the user selects DVD as the output media type.

 

  • Queue Type: Fax

    • Queue
      Specifies a name for the RightFax/ZetaFax queue.

    • Password
      Specifies the password for the RightFax/ZetaFax queue.

 

  • Queue Type: Email

    • SMTP Host
      Specifies the name of the host or IP Address.

    • SMTP Port
      Specifies the number of the port. The default is 25.

    • Email Size Limit
      Specifies the maximum size in megabytes (MB) of the email. The default is 25MB. If the email size exceeds this default, the job will fail.  

    • From Email Address
      Specifies the sending email address.

    • From Name
      Specifics who the email is from.

    • Email Subject
      Indicates the subject of the email.  The field populates automatically if default information has been configured by the system administrator.  If your email server supports encrypted email delivery, please contact your mail server administrator for configuration details.  If the mail server supports filter-based encryption, type SECURE: and the applicable text - for example, "SECURE: Medical Records Documents."  The user cannot change this text.

    • Email Body
      Specifies any information related to the email. The field populates automatically if default information has been configured by the system administrator.  

    • Override subject and body?
      Allows the user to overwrite the preconfigured subject and content body at the time of output.

    • Password Required for PDF Files
      Specifies any protection for the file.  If No is selected, the system allows the user to password protect the file when it is output.  If Yes is selected, the system protects the document.

      Documents include:

      • MPF Documents

      • Supplemental Documents

      • Invoices

      • Pre-Bills

      • Letters

      • Attachments

    • Default Password
      Indicates the password required for the PDF Files.

    • Mime Type
      Specifies the email security setting.
